                           1. 리포트 시스템 변경
================================================================================

[배경]
- 현재: 각 접수 건마다 개별적으로 일일 기록 입력
- 문제: 80개 업체면 80개 엑셀을 각각 업로드해야 함

[요구사항]
1. 관리자가 전체 데이터가 포함된 엑셀 파일 1개를 업로드
2. 시스템이 "업체명" 기준으로 자동 분류
3. 각 접수 건에 해당 데이터 자동 매칭
4. 클라이언트는 본인 접수 건에서 "리포트 다운로드" 클릭
5. 해당 접수 건의 데이터만 분리된 엑셀 다운로드

[추가 요구사항]
- 매일 업데이트 가능 (100건 중 1건만 완료되어도 리포트 다운로드 가능)
- 클라이언트가 매일 진행 현황 확인 가능

[비용]
- 원래 50만원 → 무료 (대표님 지시)

                    3. 카카오 로그인 온보딩 추가
================================================================================

[배경]
- 현재 두 가지 가입 방식 존재:
  1. 관리자가 직접 아이디/비번 생성 → 광고주/대행사 구분 가능
  2. 카카오 로그인 → 광고주/대행사 구분 불가

[요구사항]

- 선택 항목:
  1. 대행사 / 개별 광고주 선택
  2. 사업자 정보 입력 (기존 마이페이지 정보와 동일)
- 정보 입력 완료 후 서비스 이용 가능
- "사업자 정보를 등록해야 포인트 충전 및 이용이 가능합니다" 안내

[플로우]
1. 카카오 로그인 클릭
2. 카카오 인증 완료
3. [온보딩 팝업]
   - 대행사 / 개별 광고주 선택
   - 사업자 정보 입력
4. 정보 입력 완료 → 서비스 이용 가능

================================================================================
                    4. AI 리뷰 생성 기능 개선
================================================================================

[4-1. 기본 프롬프트 변경]
--------------------------------------------------------------------------------
현재 사용 중인 ChatGPT 프롬프트를 기본값으로 설정:

"""
🎯 리뷰 생성 통합 가이드

✅ 기본 톤 & 구성
- 말투: 20~50대 자연스럽게 섞기
- MZ식 표현(예: 찐맛, 미쳤다, 인정, 밥도둑 등)도 일부 포함
- 자연어 기반 실제 후기 말투 중심
- 상황: 데이트 / 친구 / 가족 / 회식 / 퇴근 / 여행 등 랜덤
- 분량: 1줄 / 2줄 / 3줄 랜덤 구성
- 이모티콘 비율: 5% (😊🔥💯😋🫶 등 자연스럽게 가끔만 등장)
- 맘카페톤 제외 (리얼 후기체 / 홍보 느낌 X)
- 큰따옴표 절대 금지
- 리뷰 간 행간 띄우기 없음 (한 줄씩 나열)
- 복붙 가능한 텍스트 포맷

✅ 문장 끝맺음 다양화
~~했음 / ~~좋았음 / ~~완벽했어요 / ~~괜찮았어요 / ~~추천합니다 /
~~인정 / ~~미쳤다 / ~~레전드 / ~~굿 / ~~가야 함 / ~~또 올 듯 /
~~찐 / ~~대박 / ~~개존맛 / ~~맘에 들었음
"""

[4-2. 업체 정보 입력 방식]
--------------------------------------------------------------------------------
- 자동 검색 X (유명하지 않은 업체는 검색 부정확)
- 텍스트로 업체 정보 수동 입력 (이미 구현됨)
- 프롬프트 형식: "{업체명} 검색해보고 아래 가이드 참고해서..."
  → "{업체명}"은 접수된 업체명으로 자동 변경

[4-3. 글자수 비율 설정 UI 변경]
--------------------------------------------------------------------------------
현재: 슬라이더(그래프 끌어당기기)로 비율 설정
변경: 숫자 직접 입력 방식

- 짧은글: 60자 이내 → [ ] 개
- 중간글: 100자 이내 → [ ] 개
- 긴글: 150자 이내 → [ ] 개
- 합계가 총 주문 개수와 일치해야 함

[4-4. 생성된 리뷰 표시 방식]
--------------------------------------------------------------------------------
현재: 개별 카드 형태로 표시 (각각 복사해야 함)
변경: 텍스트 목록 형태로 표시 (한 번에 복사 가능)

예시 출력:
"""
여행 중 들렀는데 연포탕 첫 한입에 속이 편안해져서 괜히 기분 좋아졌습니다…
목포항 걷다가 들어갔는데 연포탕 국물이 깔끔해 계속 손이 갔고 반찬도 정갈해 만족스러웠어요! 분위기까지 조용해서 더 좋았습니다 ^^
비가 갑자기 쏟아져 들렀는데 따뜻한 국물이 몸을 바로 풀어줘서 여행 중 잠깐 숨 돌리기 좋았어요 또 올 듯~
...
"""

[4-5. 복사 및 다운로드 기능 추가]
--------------------------------------------------------------------------------
- [전체 복사] 버튼: 생성된 리뷰 전체를 클립보드에 복사
- [엑셀 다운로드] 버튼: 생성된 리뷰를 엑셀 파일로 다운로드
  → 실행직원에게 바로 전달 가능한 형태

================================================================================
                    5. 카카오맵 리뷰 워크플로우 변경
================================================================================

[5-1. 현재 플로우 (AS-IS)]
--------------------------------------------------------------------------------
1. 클라이언트 접수
2. 관리자가 AI 리뷰 생성
3. 관리자가 개별 리뷰에 이미지 매칭 후 업로드
4. 클라이언트가 개별 리뷰 카드 확인
5. 각 리뷰마다 승인/수정요청
6. 개별 피드백 처리

[5-2. 변경 플로우 (TO-BE)]
--------------------------------------------------------------------------------
1. 클라이언트 접수 (이미지는 이메일로 별도 전송 - 아래 6번 참고)
2. 관리자가 접수 확인
3. 관리자가 AI 리뷰 생성 버튼 클릭
4. 생성된 리뷰 텍스트 목록 확인 (개별 카드 X)
5. [광고주 컨펌 요청] 버튼 클릭 → 클라이언트에게 전달
6. 클라이언트가 전체 리뷰 텍스트 확인
7. [전체 승인] 또는 [피드백 보내기] (개별 승인 X)
8. 피드백 시 관리자가 수정 후 재전달
9. 최종 승인 후 작업 진행

[5-3. 피드백 구조 변경]
--------------------------------------------------------------------------------
현재: 개별 리뷰마다 피드백 가능
변경: 전체 리뷰에 대해 한 번의 피드백만

- 클라이언트는 전체 리뷰 목록을 보고
- 전체에 대한 의견을 한 번에 작성
- 관리자가 피드백 반영 후 재전달
