애드센스 마케팅 상품 접수 시스템 - 관리자 기본 운영 가이드

이 문서는 관리자가 시스템을 효율적으로 운영하기 위한 기본 가이드입니다.


1. 관리자 로그인
==================================================

초기 설정:
1. 브라우저에 시스템 주소를 입력합니다
2. "관리자" 탭 선택
3. 아이디: admin (기본값)
4. 비밀번호: admin123 (기본값)
5. "로그인" 버튼 클릭

보안: 기본 비밀번호는 반드시 변경하세요!


2. 거래처 추가하기
==================================================

새로운 거래처를 등록하는 방법:

1. "거래처 관리" 메뉴 클릭
2. "거래처 추가" 버튼 클릭
3. 다음을 입력합니다:

   아이디: 거래처의 로그인 아이디
   - 예: cafe_seoul_001, restaurant_123
   - 중복 불가, 영문/숫자/언더스코어 사용 가능

   비밀번호: 거래처의 로그인 비밀번호
   - 예: Pass@1234
   - 설정 후 변경 불가이므로 신중하게 설정

   회사명: 거래처의 정식 회사명
   - 예: 강남 커피숍, 서울 피자 전문점

   초기 포인트: 거래처가 처음 받을 포인트
   - 예: 50000 (계약금액에 따라 결정)

4. "거래처 추가" 버튼 클릭
5. 목록에 새 거래처 추가됨

거래처에 알려줄 정보:
- 아이디
- 비밀번호 (안전한 채널로 전달)
- 시스템 접속 주소
- 초기 포인트 금액


3. 거래처 정보 수정
==================================================

거래처 정보를 변경하는 방법:

1. "거래처 관리" 페이지 열기
2. 수정할 거래처의 "수정" 버튼 클릭
3. 다음을 수정할 수 있습니다:
   - 아이디: 새 아이디로 변경 (중복 불가)
   - 회사명: 회사 이름 변경
   - 상태: 활성/비활성 전환
4. "저장" 클릭

주의:
- 아이디 변경 시 거래처에 알리기
- 비활성은 거래처가 로그인 불가능
- 비활성은 비활성화 전 거래처에 알리기


4. 비밀번호 초기화
==================================================

거래처가 비밀번호를 잊었을 때:

1. "거래처 관리" 페이지 열기
2. 대상 거래처의 "비밀번호 초기화" 버튼 클릭
3. 새로운 임시 비밀번호 입력 (예: TempPass123)
4. "초기화" 버튼 클릭
5. 임시 비밀번호를 거래처에 전달 (안전한 채널로)

주의: 안전한 채널(전화, 보안 메시지)로 전달하세요


5. 상품 가격 설정하기
==================================================

거래처별로 상품 가격을 설정하는 방법:

1. "거래처 관리" 열기
2. 대상 거래처의 "가격 설정" 버튼 클릭
3. 4가지 상품의 가격을 입력합니다:

   플레이스 유입: (예) 5000
   영수증 리뷰: (예) 4000
   카카오맵 리뷰: (예) 6000
   블로그 배포: (예) 10000

4. 각 상품의 "표시" 체크박스:
   - 체크: 거래처가 볼 수 있음
   - 미체크: 거래처가 볼 수 없음

5. "저장" 클릭

전략:
- 모든 상품 제공: 모두 체크
- 선별 제공: 원하는 상품만 체크
- 거래처별 차등: 규모에 따라 다른 가격


6. 포인트 관리
==================================================

거래처의 포인트를 관리하는 방법:

1. "포인트 관리" 메뉴 클릭
2. 모든 거래처의 포인트 현황 표시
3. 조정할 거래처의 "관리" 버튼 클릭
4. 다음을 입력합니다:

   변경 유형:
   - 충전: 포인트 추가
   - 차감: 포인트 회수
   - 환불: 포인트 돌려줌

   포인트: 변경 금액 (예: 50000)

   사유: 변경 이유 (예: "계약금 입금")

5. "저장" 클릭

언제 사용:
- 충전: 계약금 입금, 추가 주문 시
- 환불: 오류로 차감된 경우, 서비스 미제공
- 차감: 거의 사용하지 않음 (환불로 처리)


7. 접수 내역 확인하기
==================================================

모든 거래처의 신청 기록을 보는 방법:

1. "접수 내역" 메뉴 클릭
2. 모든 신청이 표로 표시됨:
   - 날짜: 신청한 시간
   - 거래처: 회사명
   - 상품: 상품 종류
   - 수량: 신청 개수
   - 포인트: 사용한 포인트

필터링:
- 거래처 선택: 특정 거래처의 신청만 보기
- 상품 선택: 특정 상품의 신청만 보기
- 날짜 범위: 기간을 정해서 보기

데이터 다운로드:
1. 필터링 완료
2. "엑셀 다운로드" 버튼 클릭
3. 파일이 컴퓨터에 저장됨
4. 엑셀에서 추가 분석 가능


8. AS(서비스요청) 처리
==================================================

거래처의 문제 신청을 처리하는 방법:

1. "AS 신청" 메뉴 클릭
2. 거래처의 문제 신청 목록 표시
3. 각 신청을 읽고 다음을 확인합니다:
   - 문제가 무엇인가?
   - 누가 신청했는가?
   - 언제 신청했는가?

주요 문제 유형:

로그인 문제:
- 아이디와 비밀번호 확인해주기
- 필요하면 비밀번호 초기화
- 브라우저 캐시 삭제 안내

포인트 부족:
- 현재 포인트 확인해주기
- 필요하면 포인트 충전

신청 오류:
- 신청 기록 확인
- 포인트가 차감되었으면 환불

서비스 미제공:
- 신청 내용 확인
- 이유 파악
- 거래처에 설명
- 필요하면 환불

처리 절차:
1. 거래처와 연락 (전화/이메일)
2. 문제 확인
3. 해결 조치 (포인트 환불 등)
4. 해결 완료 후 상태 변경


9. 월간 체크리스트
==================================================

관리자가 매월 확인할 사항:

월초:
□ 지난달 신청 통계 확인
□ 거래처별 포인트 현황 파악
□ 비활성 거래처 확인

월중:
□ 일일 신청 현황 모니터링
□ 포인트 관련 문의 처리
□ 신규 거래처 온보딩

월말:
□ 월간 신청 현황 분석
□ 거래처별 매출 정산
□ 다음달 목표 설정

분기별:
□ 전체 매출 분석
□ 상품별 성과 평가
□ 가격 정책 검토


10. 보안 및 주의사항
==================================================

시스템 보안:
- 기본 비밀번호는 반드시 변경
- 작업 후 항상 로그아웃
- 불필요한 창은 닫기
- 공용 컴퓨터 사용 금지

거래처 정보 보호:
- 아이디/비밀번호는 기밀 유지
- 포인트 정보는 필요한 사람에게만 공개
- 신청 정보는 기밀 문서로 취급

흔한 실수:
1. 거래처 삭제: 복구 불가능하므로 신중하게
2. 포인트 중복 차감: 신청 기록 확인 후 조정
3. 가격 설정 오류: 설정 후 검토
4. 비밀번호 초기화 후 전달 안 함: 즉시 거래처에 연락


11. 자주 묻는 질문
==================================================

Q: 모든 거래처에 같은 가격을 일괄 적용할 수 있나요?
A: 불가능합니다. 각 거래처마다 개별 설정해야 합니다.

Q: 거래처를 실수로 삭제했습니다. 복구 가능한가요?
A: 복구 불가능합니다. 같은 정보로 새 계정을 만들어야 합니다.

Q: 신청 기록을 모두 엑셀로 받을 수 있나요?
A: 가능합니다. "접수 내역"에서 "엑셀 다운로드" 버튼 클릭하면 됩니다.

Q: 포인트가 잘못 차감되었을 때는?
A: "포인트 관리"에서 "환불"을 선택해서 포인트를 돌려주세요.

Q: 거래처가 신청을 취소하고 싶어요.
A: 신청은 취소 불가능하지만 포인트 환불은 가능합니다.
