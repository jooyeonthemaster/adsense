제1장: 시스템 개요 및 로그인
============================================================================

==================================================
1-1. 시스템 소개
==================================================

본 시스템은 애드센스 마케팅 상품의 접수와 관리를 위한 플랫폼입니다.

【주요 기능】
- 거래처별 상품 접수 관리
- 포인트 기반 결제 시스템
- 실시간 데이터 분석
- AS 요청 처리
- 리포트 관리

【지원 상품 (4가지 고정)】
1. 플레이스 유입: 네이버 플레이스 유입 서비스
2. 영수증 리뷰: 영수증 기반 리뷰 작성
3. 카카오맵 리뷰: 카카오맵 리뷰 작성
4. 블로그 배포: 블로그 콘텐츠 배포

【시스템 특징】
- Next.js 기반 반응형 웹 애플리케이션
- 모바일/태블릿/데스크탑 모두 지원
- 실시간 데이터 동기화
- 보안이 강화된 인증 시스템


==================================================
1-2. 관리자 로그인 방법
==================================================

【접속 방법】

1단계: 웹 브라우저 실행
   - 권장 브라우저: Chrome, Edge, Safari, Firefox
   - 시스템 주소를 주소창에 입력합니다

2단계: 로그인 페이지 확인
   - 화면 상단에 "거래처" / "관리자" 두 개의 탭이 보입니다
   - "관리자" 탭을 클릭합니다

3단계: 관리자 계정 로그인
   - 아이디 입력: admin (기본값)
   - 비밀번호 입력: admin123 (기본값)
   - "로그인" 버튼 클릭

4단계: 로그인 완료
   - 관리자 대시보드로 자동 이동합니다
   - 왼쪽에 네비게이션 메뉴가 표시됩니다

【보안 주의사항】
⚠️ 중요: 기본 비밀번호는 반드시 변경하세요!
⚠️ 작업 완료 후 반드시 로그아웃하세요!
⚠️ 공용 컴퓨터에서는 사용하지 마세요!


==================================================
1-3. 관리자 화면 구성
==================================================

관리자 화면은 크게 3개의 영역으로 구성됩니다:

【1. 네비게이션 영역 (왼쪽)】
- 시스템 로고 및 이름
- 메뉴 목록
- 로그아웃 버튼

【2. 헤더 영역 (상단)】
- 현재 페이지 제목
- 액션 버튼들

【3. 컨텐츠 영역 (중앙)】
- 주요 내용 표시
- 데이터 테이블
- 통계 차트

【모바일 화면】
- 모바일에서는 햄버거 메뉴(☰)를 터치하여 네비게이션 열기
- 카드 형식으로 정보 표시
- 스크롤을 통한 탐색


==================================================
1-4. 네비게이션 메뉴 설명
==================================================

왼쪽 네비게이션 메뉴는 다음과 같이 구성됩니다:

【대시보드】 (아이콘: 레이아웃)
- 경로: /admin
- 기능: 시스템 전체 현황을 한눈에 확인
- 언제 사용: 처음 로그인했을 때, 빠른 현황 파악 시

【데이터 분석】 (아이콘: 차트)
- 경로: /admin/analytics
- 기능: 상세한 통계 및 분석 데이터 조회
- 언제 사용: 월간 보고서 작성, 트렌드 분석 시

【거래처 관리】 (아이콘: 사용자)
- 경로: /admin/clients
- 기능: 거래처 계정 생성, 수정, 삭제, 가격 설정
- 언제 사용: 신규 거래처 등록, 정보 수정 필요 시

【접수 내역】 (아이콘: 파일)
- 경로: /admin/submissions
- 기능: 모든 거래처의 상품 접수 내역 조회 및 관리
- 언제 사용: 접수 처리, 진행 상황 확인 시

【포인트 관리】 (아이콘: 달러)
- 경로: /admin/points
- 기능: 포인트 거래 내역 조회
- 언제 사용: 포인트 사용 내역 확인, 회계 정산 시

【AS 관리】 (아이콘: 설정)
- 경로: /admin/as-requests
- 기능: 거래처의 AS 요청 확인 및 처리
- 언제 사용: 문제 발생 시, 고객 불만 처리 시

【로그아웃】 (하단)
- 기능: 관리자 계정에서 로그아웃
- 언제 사용: 작업 완료 후, 자리를 비울 때

제2장: 관리자 대시보드
============================================================================

==================================================
2-1. 대시보드 개요
==================================================

대시보드는 관리자가 로그인 후 가장 먼저 보는 화면입니다.
시스템 전체 현황을 한눈에 파악할 수 있도록 설계되었습니다.

【화면 구성】
1. 페이지 헤더
   - 제목: "관리자 대시보드"
   - 부제: "애드센스 마케팅 플랫폼 관리 시스템"

2. 통계 카드 (4개)
   - 총 거래처
   - 대기 중인 접수
   - AS 신청
   - 총 포인트

3. 최근 접수 내역 카드
   - 최근 24시간 내 접수 10건
   - 대기중 필터 버튼


==================================================
2-2. 통계 카드 이해하기
==================================================

상단에 4개의 통계 카드가 가로로 배열되어 있습니다.
각 카드는 중요한 지표를 실시간으로 표시합니다.

【카드 1: 총 거래처】
- 아이콘: 사용자 (파란색 그라데이션 배경)
- 표시 내용: 현재 등록된 거래처 총 개수
- 설명: "활성 거래처 수"
- 예시: "23"
- 의미: 시스템에 등록된 전체 거래처 수

활용 방법:
- 거래처가 증가하는 추세인지 확인
- 영업 목표 달성률 파악
- 월별 증가율 비교

【카드 2: 대기 중인 접수】
- 아이콘: 파일 (청록색 그라데이션 배경)
- 표시 내용: 처리 대기 중인 접수 건수
- 설명: "처리 대기 중"
- 예시: "8"
- 의미: 아직 처리하지 않은 접수

활용 방법:
- 매일 아침 확인하여 처리 계획 수립
- 숫자가 많으면 우선 처리 필요
- 팀원에게 작업 분배

【카드 3: AS 신청】
- 아이콘: 경고 (보라색 그라데이션 배경)
- 표시 내용: 처리 대기 중인 AS 요청 수
- 설명: "처리 대기 중"
- 예시: "3"
- 의미: 거래처의 문제 신고/요청

활용 방법:
- 빠른 응대가 필요한 항목
- 매일 최소 2회 이상 확인 권장
- AS 처리 우선순위 결정

【카드 4: 총 포인트】
- 아이콘: 달러 (초록색 그라데이션 배경)
- 표시 내용: 모든 거래처의 보유 포인트 합계
- 설명: "전체 거래처 보유"
- 예시: "2,450,000"
- 의미: 시스템 내 순환하는 총 포인트

활용 방법:
- 전체 자금 규모 파악
- 월별 증감 추세 분석
- 재무 보고서 작성 시 활용

【카드 디자인 특징】
- 각 카드마다 고유한 색상 그라데이션
- 호버 시 부드러운 애니메이션 효과
- 숫자가 크면 자동으로 천 단위 콤마 표시


==================================================
2-3. 최근 접수 내역 확인
==================================================

대시보드 하단에는 "최근 접수 내역" 카드가 있습니다.
최근 24시간 동안의 접수를 최대 10건까지 표시합니다.

【카드 헤더】
- 좌측: 제목 "최근 접수 내역" + 활동 아이콘
- 우측: "대기중만" / "전체 보기" 필터 버튼

【접수 카드 정보】 (각 접수마다 별도 카드)
1. 거래처 정보
   - 거래처명 (상단, 큰 글씨)
   - 업체명 (하단, 작은 글씨)

2. 포인트 정보
   - 사용 포인트 (우측 상단, 주황색 강조)
   - 예: "15,000 P"

3. 상품 및 상태 배지
   - 상품 유형 배지 (테두리 스타일)
     * 플레이스 유입
     * 영수증 리뷰
     * 카카오맵 리뷰
     * 블로그 배포
   
   - 상태 배지 (색상 구분)
     * 대기중 (회색 테두리)
     * 진행중 (파란색)
     * 완료 (초록색)
     * 취소 (빨간색)

4. 접수 일시
   - 형식: "11월 4일 14:23"
   - 하단에 작은 글씨로 표시

【정보 읽는 방법】
예시 접수:
┌─────────────────────────────────────┐
│ 강남 커피숍            15,000 P    │
│ 스타벅스 강남점                    │
│                                     │
│ 🏷 플레이스 유입  🔵 진행중        │
│                                     │
│ 11월 4일 14:23                     │
└─────────────────────────────────────┘

해석:
- "강남 커피숍" 거래처가
- "스타벅스 강남점"에 대한
- "플레이스 유입" 상품을
- 15,000 포인트로 신청했고
- 현재 "진행중" 상태


==================================================
2-4. 대기중 필터 사용법
==================================================

최근 접수 내역 카드의 우측 상단에 필터 버튼이 있습니다.

【버튼 위치】
- 카드 제목 옆, 우측 상단
- 필터 아이콘(깔때기 모양) + 텍스트

【사용 방법】

1단계: 버튼 클릭
   - 기본 상태: "대기중만" (비활성 스타일)
   - 클릭하면 파란색으로 변경

2단계: 필터 적용 확인
   - 활성화: "전체 보기" 표시 (파란색 배경)
   - 비활성화: "대기중만" 표시 (회색 테두리)

3단계: 결과 확인
   - 활성화 시: 대기중 상태만 표시
   - 비활성화 시: 모든 상태 표시

【활용 시나리오】

시나리오 1: 아침 업무 시작
- 대기중 필터 활성화
- 오늘 처리해야 할 접수만 확인
- 우선순위 결정

시나리오 2: 전체 현황 파악
- 필터 비활성화
- 모든 접수 확인
- 진행 상황 모니터링

【모바일 화면】
- 모바일에서는 접수가 세로로 쌓여 표시
- 스크롤하여 모든 접수 확인 가능
- 필터 버튼은 동일하게 작동

【데스크탑 화면】
- 접수가 그리드 형태로 표시 (3열)
- 한 화면에 더 많은 정보 표시
- 더 편한 비교 가능

제3장: 데이터 분석 (애널리틱스)
============================================================================

==================================================
3-1. 애널리틱스 페이지 개요
==================================================

데이터 분석 페이지는 시스템의 모든 데이터를 시각화하여 보여줍니다.
경영 의사결정과 보고서 작성에 활용할 수 있는 핵심 페이지입니다.

【페이지 접근 방법】
1. 좌측 네비게이션에서 "데이터 분석" 클릭
2. 또는 주소: /admin/analytics

【페이지 구성】
- 헤더: "데이터 애널리틱스" 제목
- 부제: "실시간 거래량 분석 및 통계 대시보드"
- 탭 메뉴: 5개 탭으로 구성
- 컨텐츠 영역: 차트와 통계

【5개 탭 구성】
1. 전체 개요: 전반적인 현황 한눈에
2. 일간 분석: 오늘과 어제 비교
3. 주간 분석: 이번 주와 지난 주 비교
4. 월간 분석: 이번 달과 지난 달 비교
5. 상품 분석: 상품별 상세 통계

【차트 종류】
- 라인 차트: 시간별 추이
- 막대 차트: 비교 데이터
- 영역 차트: 누적 추세
- 원형 차트: 비율 분석

【업데이트 주기】
- 실시간 자동 갱신
- 페이지 새로고침 시 최신 데이터 반영


==================================================
3-2. 전체 개요 탭
==================================================

전체 개요 탭은 가장 포괄적인 정보를 제공합니다.

【실시간 KPI 카드 (4개)】

1. 오늘 접수
   - 아이콘: 쇼핑카트 (청록색 그라데이션)
   - 표시: 오늘 접수된 건수
   - 비교: 어제 대비 증감률
   - 예시: "15건" / "+25.3% ↗"
   - 색상: 증가 = 초록색, 감소 = 빨간색

2. 오늘 매출
   - 아이콘: 달러 (파란색 그라데이션)
   - 표시: 오늘 사용된 총 포인트
   - 비교: 어제 대비 증감률
   - 예시: "450,000 P" / "+18.7% ↗"

3. 신규 거래처
   - 아이콘: 사용자 (보라색 그라데이션)
   - 표시: 오늘 신규 등록 거래처 수
   - 비교: 어제 대비 증감률
   - 예시: "2개" / "+100% ↗"

4. 총 거래처
   - 아이콘: 사용자 (초록색 그라데이션)
   - 표시: 전체 거래처 수
   - 비교: 안정적 지표 (변동 없음)
   - 예시: "23개" / "0% ─"

【주요 지표 카드 (3개)】

1. 대기 중 접수
   - 처리가 필요한 접수 건수
   - 큰 숫자로 강조
   - 하단: "처리 필요" 안내

2. 완료율
   - 전체 접수 대비 완료된 비율
   - 퍼센트로 표시
   - 하단: "완료 / 전체" 건수

3. AS 신청
   - 대기중인 AS 요청 수
   - 즉시 처리 필요한 항목
   - 하단: "대기 중" 안내

【최근 30일 접수 추이 차트】
- 유형: 라인 차트 (이중 축)
- 좌측 축: 접수 건수 (파란 선)
- 우측 축: 포인트 사용 (초록 선)
- X축: 날짜 (MM-DD 형식)
- 기능:
  * 마우스 호버 시 상세 정보
  * 날짜별 정확한 수치 확인
  * 추세 파악 가능

【상품별 분포 차트】
- 유형: 원형 차트 (파이 차트)
- 위치: 좌측 하단
- 표시: 4가지 상품의 접수 비율
  * 플레이스 유입: 청록색
  * 영수증 리뷰: 파란색
  * 카카오맵 리뷰: 보라색
  * 블로그 배포: 초록색
- 기능:
  * 각 섹션에 비율 표시
  * 마우스 호버 시 정확한 건수
  * 범례로 색상 구분

【TOP 10 거래처 차트】
- 유형: 가로 막대 차트
- 위치: 우측 하단
- 정렬: 접수 건수 많은 순
- 표시: 상위 10개 거래처
- Y축: 거래처명 (최대 100px 폭)
- X축: 접수 건수
- 색상: 파란색 막대
- 기능:
  * 주요 거래처 한눈에 파악
  * 매출 기여도 분석
  * VIP 거래처 관리

【인사이트 지표 (4개)】

1. 평균 처리 시간
   - 단위: 일(day)
   - 의미: 접수부터 완료까지 평균 소요 기간
   - 활용: 업무 효율성 측정

2. AS 발생률
   - 단위: 퍼센트(%)
   - 의미: 전체 접수 대비 AS 발생 비율
   - 활용: 서비스 품질 지표

3. 포인트 회전율
   - 단위: 퍼센트(%)
   - 의미: 포인트 사용 대비 충전 비율
   - 활용: 자금 유동성 파악

4. 활성 거래처
   - 형식: "N / M 개"
   - N: 최근 활동한 거래처
   - M: 전체 거래처
   - 활용: 거래처 참여율 측정


==================================================
3-3. 일간 분석 탭
==================================================

일간 분석 탭은 오늘과 어제를 비교하여 단기 트렌드를 파악합니다.

【주요 통계 카드 (3개)】

1. 오늘 접수
   - 메인 숫자: 오늘 접수 건수
   - 부제: "어제: N건"
   - 증감: 색상과 화살표로 표시
   - 아이콘: 캘린더

2. 오늘 매출
   - 메인 숫자: "N P" (포인트)
   - 부제: "어제: N P"
   - 증감률 표시
   - 아이콘: 달러

3. 신규 거래처
   - 메인 숫자: 신규 등록 수
   - 부제: "어제: N개"
   - 증감 표시
   - 아이콘: 사용자

【최근 30일 일간 추이 차트】
- 유형: 영역 차트 (Area Chart)
- 데이터: 최근 30일간의 일별 접수 건수
- 특징:
  * 그라데이션 효과 (상단 진함, 하단 투명)
  * 부드러운 곡선
  * 추세 시각화에 최적
- X축: 날짜 (MM-DD)
- Y축: 접수 건수
- 색상: 파란색 그라데이션

활용 예시:
- 주말과 평일 접수량 비교
- 특정일 급증/급감 원인 분석
- 계절적 패턴 발견

【시간대별 접수 패턴 차트】
- 유형: 세로 막대 차트
- 데이터: 0시~23시 시간대별 접수 현황
- X축: 시간 (0~23)
- Y축: 접수 건수
- 색상: 파란색 막대

패턴 분석:
- 오전 시간대 (9시~12시): 주로 높음
- 점심 시간 (12시~14시): 보통 낮음
- 오후 시간대 (14시~18시): 높음
- 저녁 시간대 (18시~): 점차 감소

활용 방법:
- 피크 타임 파악
- 인력 배치 계획
- 시스템 부하 예측


==================================================
3-4. 주간 분석 탭
==================================================

주간 분석 탭은 이번 주와 지난 주를 비교합니다.

【주요 통계 카드 (3개)】

1. 이번 주 접수
   - 메인: 이번 주 총 접수
   - 부제: "지난 주: N건"
   - 증감률 표시

2. 이번 주 매출
   - 메인: 이번 주 포인트 사용
   - 부제: "지난 주: N P"
   - 증감률 표시

3. 일평균 접수
   - 메인: 평균 일일 접수
   - 부제: "지난 주: N건"
   - 소수점 첫째 자리까지

【주간 추이 차트】
- 유형: 이중 막대 차트
- 데이터: 최근 주별 접수 및 포인트
- 막대 1: 접수 건수 (파란색)
- 막대 2: 포인트 사용 (초록색)
- X축: 주 (YYYY-MM-DD 형식)
- Y축: 값 (자동 스케일)
  * 1000 이상: "1K"
  * 1,000,000 이상: "1M"

활용 방법:
- 주간 성과 비교
- 월간 목표 진척도 확인
- 주별 패턴 분석

==================================================
4-1. 거래처 목록 화면
==================================================

거래처 관리는 시스템의 핵심 기능입니다.
모든 거래처 계정을 생성하고 관리합니다.

【페이지 접근】
- 좌측 네비게이션 "거래처 관리" 클릭
- 주소: /admin/clients

【페이지 헤더】
- 제목: "거래처 관리"
- 부제: "거래처 계정을 생성하고 관리합니다"
- 우측: "거래처 추가" 버튼 (파란색, + 아이콘)

【필터 영역 (3개)】

1. 검색 필터
   - 입력란: 텍스트 입력
   - 플레이스홀더: "회사명, 담당자, 아이디..."
   - 기능: 실시간 검색 (타이핑하면 즉시 필터링)
   - 검색 대상:
     * 회사명
     * 담당자명
     * 아이디 (username)

2. 상태 필터
   - 드롭다운 선택
   - 옵션:
     * 전체 (기본값)
     * 활성
     * 비활성
   - 기능: 상태별 거래처 필터

3. 정렬 필터
   - 드롭다운 선택
   - 옵션:
     * 최신순 (기본값)
     * 오래된순
     * 가나다순
     * 가나다역순
     * 포인트 많은순
     * 포인트 적은순
   - 기능: 다양한 기준으로 정렬

【모바일 화면】
거래처가 카드 형태로 표시:
- 각 카드에 거래처 정보 요약
- 스크롤하여 탐색
- 주요 버튼들 카드 하단 배치

【데스크탑 화면】
거래처가 테이블 형태로 표시:

테이블 컬럼 (7개):
1. 아이디: 로그인 아이디
2. 회사명: 거래처 회사명
3. 담당자: 담당자 이름
4. 연락처: 전화번호
5. 포인트: 보유 포인트 (굵게 강조)
6. 상태: 활성/비활성 배지
7. 작업: 액션 버튼들

【각 거래처 행의 작업 버튼 (5개)】

1. 가격 설정 (톱니바퀴 아이콘)
   - 기능: 거래처별 상품 가격 설정
   - 경로: /admin/clients/[id]/pricing

2. 포인트 관리 (달러 아이콘)
   - 기능: 포인트 충전/차감
   - 다이얼로그 팝업

3. 비밀번호 재설정 (열쇠 아이콘)
   - 기능: 거래처 비밀번호 변경
   - 다이얼로그 팝업

4. 거래처 수정 (연필 아이콘)
   - 기능: 거래처 정보 수정
   - 다이얼로그 팝업

5. 거래처 삭제 (휴지통 아이콘, 빨간색)
   - 기능: 거래처 삭제
   - 다이얼로그 팝업 (확인 필요)


==================================================
4-2. 거래처 검색 및 필터링
==================================================

거래처가 많을 때 효율적으로 찾는 방법입니다.

【검색 기능 사용법】

1단계: 검색어 입력
   - 검색 필터 입력란 클릭
   - 회사명, 담당자명, 또는 아이디 입력
   - 예: "강남" 입력

2단계: 실시간 결과 확인
   - 타이핑과 동시에 목록 필터링
   - 매칭되는 거래처만 표시
   - 매칭 항목 수 카운트 업데이트

3단계: 검색 초기화
   - 검색어를 모두 지우면
   - 전체 목록으로 복원

검색 팁:
- 부분 검색 지원: "강남" 입력 시 "강남 커피", "서울 강남점" 모두 검색
- 대소문자 구분 없음
- 공백 포함 검색 가능

【상태 필터 사용법】

1단계: 상태 드롭다운 클릭
   - "상태" 라벨 아래 드롭다운

2단계: 원하는 상태 선택
   - 전체: 모든 거래처
   - 활성: 로그인 가능한 거래처만
   - 비활성: 로그인 불가능한 거래처만

3단계: 필터 적용 확인
   - 선택한 상태의 거래처만 표시
   - 카운트 업데이트

활용 시나리오:
- 활성 거래처만 보기: 실제 영업 중인 거래처 파악
- 비활성 거래처 확인: 재활성화 대상 검토

【정렬 기능 사용법】

1단계: 정렬 드롭다운 클릭
   - "정렬" 라벨 아래 드롭다운

2단계: 정렬 기준 선택
   - 최신순: 최근 등록된 거래처부터
   - 오래된순: 오래된 거래처부터
   - 가나다순: 회사명 ㄱㄴㄷ 순
   - 가나다역순: 회사명 역순
   - 포인트 많은순: 포인트 보유 많은 순
   - 포인트 적은순: 포인트 보유 적은 순

3단계: 정렬 결과 확인
   - 선택한 기준으로 목록 재정렬
   - 즉시 반영

활용 시나리오:
- 포인트 많은순: VIP 거래처 관리
- 가나다순: 특정 거래처 빠른 찾기
- 최신순: 신규 거래처 모니터링

【필터 조합 사용】
모든 필터를 동시에 사용 가능:

예시 1: 활성 거래처 중 포인트 많은 순
1. 상태 필터: "활성" 선택
2. 정렬: "포인트 많은순" 선택
→ 결과: 활성 거래처만 포인트 많은 순으로 표시

예시 2: "강남" 포함 거래처를 가나다순
1. 검색: "강남" 입력
2. 정렬: "가나다순" 선택
→ 결과: 강남이 포함된 거래처를 가나다순으로 표시


==================================================
4-3. 거래처 추가하기
==================================================

새로운 거래처를 등록하는 전체 프로세스입니다.

【추가 시작】
1. 우측 상단 "거래처 추가" 버튼 클릭
2. 다이얼로그(팝업) 창 열림

【다이얼로그 화면 구성】

제목: "거래처 추가"
설명: "새로운 거래처 계정을 생성합니다."

【입력 필드 (7개)】

1. 아이디 (필수)
   - 라벨: "아이디 *" (빨간 별표)
   - 플레이스홀더: "로그인 아이디"
   - 제약사항:
     * 영문, 숫자, 언더스코어만 사용
     * 중복 불가
     * 예: "gangnam_cafe_001"
   - 주의: 생성 후 변경 불가!

2. 비밀번호 (필수)
   - 라벨: "비밀번호 *"
   - 타입: 비밀번호 입력 (●●●●)
   - 플레이스홀더: "초기 비밀번호"
   - 제약사항:
     * 최소 4자 이상 권장
     * 안전한 비밀번호 사용
   - 주의: 거래처에게 안전하게 전달!

3. 회사명 (필수)
   - 라벨: "회사명 *"
   - 플레이스홀더: "회사명 또는 상호"
   - 예: "강남 커피숍", "서울 피자 전문점"
   - 주의: 정식 상호명 입력

4. 담당자 (선택)
   - 라벨: "담당자"
   - 플레이스홀더: "담당자 이름"
   - 예: "김철수", "이영희"

5. 연락처 (선택)
   - 라벨: "연락처"
   - 플레이스홀더: "010-1234-5678"
   - 형식: 하이픈(-) 포함 권장

6. 이메일 (선택)
   - 라벨: "이메일"
   - 타입: 이메일
   - 플레이스홀더: "example@company.com"
   - 검증: 이메일 형식 자동 확인

7. 초기 포인트 (선택)
   - 라벨: "초기 포인트"
   - 타입: 숫자
   - 기본값: 0
   - 플레이스홀더: "0"
   - 단위: 포인트 (P)
   - 예: 50000 입력 시 50,000 P 부여

【하단 버튼 (2개)】
1. 취소: 다이얼로그 닫기 (변경사항 없음)
2. 생성: 거래처 생성 (파란색 강조)

【입력 예시】
실제 입력 예시:

아이디: gangnam_coffee_001
비밀번호: CafePass123!
회사명: 강남 스타벅스
담당자: 김매니저
연락처: 010-1234-5678
이메일: manager@gangnamcafe.com
초기 포인트: 100000

【생성 프로세스】

1단계: 정보 입력
   - 모든 필수 필드(*) 입력
   - 선택 필드는 필요시 입력

2단계: 유효성 검증
   - 아이디 중복 확인 (자동)
   - 이메일 형식 확인 (자동)
   - 필수 필드 입력 확인

3단계: 생성 버튼 클릭
   - "생성" 버튼 클릭
   - 버튼 텍스트: "생성 중..." 으로 변경
   - 로딩 상태 표시

4단계: 결과 확인
   - 성공: 다이얼로그 닫힘, 목록에 새 거래처 추가
   - 실패: 오류 메시지 표시 (빨간색 박스)
     * "이미 사용 중인 아이디입니다"
     * "거래처 생성에 실패했습니다"

【생성 후 해야 할 일】

1. 거래처에게 전달할 정보 준비:
   ┌─────────────────────────────────┐
   │ 거래처 계정 정보                 │
   ├─────────────────────────────────┤
   │ 시스템 주소: [시스템 URL]       │
   │ 아이디: gangnam_coffee_001      │
   │ 비밀번호: CafePass123!          │
   │ 초기 포인트: 100,000 P          │
   └─────────────────────────────────┘

2. 안전한 방법으로 전달:
   - 전화로 구두 전달 (권장)
   - 보안 메신저 (카카오톡, 텔레그램 등)
   - 이메일 (비밀번호는 별도 전화)
   - ❌ 공개 채팅, 문자 메시지 비권장

3. 가격 설정:
   - 거래처 목록에서 해당 거래처 찾기
   - "가격 설정" 버튼 클릭
   - 4가지 상품 가격 설정

4. 초기 테스트 요청:
   - 거래처에게 로그인 테스트 요청
   - 비밀번호 변경 권장
   - 샘플 접수 진행 안내

【주의사항】
⚠️ 아이디는 생성 후 변경 불가
⚠️ 비밀번호는 안전하게 보관 및 전달
⚠️ 초기 포인트는 계약 내용에 따라 설정
⚠️ 거래처 삭제 시 복구 불가능


==================================================
4-4. 거래처 정보 수정하기
==================================================

기존 거래처의 정보를 수정하는 방법입니다.

【수정 시작】
1. 거래처 목록에서 수정할 거래처 찾기
2. 해당 행의 "수정" 버튼(연필 아이콘) 클릭
3. 수정 다이얼로그 열림

【다이얼로그 화면】
제목: "거래처 수정"
설명: "거래처 정보를 수정합니다. (아이디: [username])"

【수정 가능 필드 (5개)】

1. 회사명 (필수)
   - 현재값: 기존 회사명 표시
   - 수정 가능

2. 담당자 (선택)
   - 현재값: 기존 담당자 표시
   - 수정 가능

3. 연락처 (선택)
   - 현재값: 기존 연락처 표시
   - 수정 가능

4. 이메일 (선택)
   - 현재값: 기존 이메일 표시
   - 수정 가능

5. 상태 (필수)
   - 드롭다운 선택
   - 옵션:
     * 활성: 로그인 가능
     * 비활성: 로그인 불가

【수정 불가 항목】
- 아이디: 변경 불가 (생성 시 고정)
- 비밀번호: 별도 "비밀번호 초기화" 기능 사용
- 포인트: 별도 "포인트 관리" 기능 사용

【하단 버튼 (2개)】
1. 취소: 변경사항 취소
2. 저장: 변경사항 저장 (파란색)

【수정 프로세스】

1단계: 정보 수정
   - 원하는 필드 수정
   - 상태 변경 (필요시)

2단계: 저장 버튼 클릭
   - "저장" 버튼 클릭
   - 버튼 텍스트: "저장 중..." 으로 변경

3단계: 결과 확인
   - 성공: 다이얼로그 닫힘, 목록 업데이트
   - 실패: 오류 메시지 표시

【상태 변경 주의사항】

활성 → 비활성 변경 시:
- 거래처가 즉시 로그인 불가
- 진행 중인 접수는 영향 없음
- 반드시 거래처에게 사전 통보!

비활성 → 활성 변경 시:
- 거래처가 다시 로그인 가능
- 즉시 적용

상태 변경 시나리오:
- 계약 종료: 비활성으로 변경
- 일시 정지: 비활성으로 변경
- 계약 재개: 활성으로 변경

【실전 예시】

시나리오 1: 담당자 변경
문제: 거래처의 담당자가 김철수 → 이영희로 변경
해결:
1. 거래처 목록에서 "수정" 클릭
2. 담당자 필드: "이영희" 입력
3. "저장" 클릭
4. 목록에서 변경 확인

시나리오 2: 계약 종료로 비활성화
문제: 계약 종료된 거래처를 비활성화
해결:
1. 거래처에게 사전 통보 (전화/이메일)
2. "수정" 클릭
3. 상태: "비활성" 선택
4. "저장" 클릭
5. 거래처 로그인 불가 확인

시나리오 3: 회사명 변경
문제: 거래처의 상호 변경 (사업자등록증 변경)
해결:
1. "수정" 클릭
2. 회사명: 새 상호 입력
3. "저장" 클릭
4. 접수 내역도 새 회사명으로 표시됨


==================================================
4-5. 거래처 삭제하기
==================================================

⚠️ 위험: 거래처 삭제는 복구 불가능합니다!

【삭제 전 확인사항】
절대 삭제하지 말아야 할 경우:
- 진행 중인 접수가 있는 경우
- 포인트 잔액이 있는 경우
- 최근 1개월 내 활동이 있는 경우

대신 할 일:
→ 상태를 "비활성"으로 변경

삭제해도 되는 경우:
- 테스트 계정
- 중복 생성된 계정
- 완전히 폐업한 거래처 (모든 거래 완료)

【삭제 프로세스】

1단계: 삭제할 거래처 확인
   - 목록에서 거래처 찾기
   - 포인트 잔액 확인
   - 최근 활동 확인

2단계: 삭제 버튼 클릭
   - 해당 행의 "삭제" 버튼 클릭
   - (빨간색 휴지통 아이콘)

3단계: 확인 다이얼로그
   제목: "거래처 삭제"
   내용:
   "정말로 [회사명] 거래처를 삭제하시겠습니까?
   
   이 작업은 되돌릴 수 없습니다.
   
   - 거래처 계정이 영구 삭제됩니다
   - 모든 접수 내역이 삭제됩니다
   - 포인트 거래 내역이 삭제됩니다
   - 복구가 불가능합니다"
   
   버튼:
   - 취소 (회색)
   - 삭제 (빨간색, 강조)

4단계: 최종 확인
   - "삭제" 버튼 클릭
   - 버튼 텍스트: "삭제 중..." 변경
   - 처리 중 표시

5단계: 삭제 완료
   - 다이얼로그 닫힘
   - 목록에서 거래처 제거
   - 복구 불가능!

【삭제 대신 비활성화 권장】

비활성화 장점:
1. 데이터 보존
   - 접수 내역 보존
   - 포인트 거래 내역 보존
   - 나중에 재활성화 가능

2. 회계 투명성
   - 과거 거래 추적 가능
   - 정산 기록 유지

3. 분쟁 대응
   - 문제 발생 시 증거 자료
   - 법적 분쟁 대비

비활성화 방법:
1. "수정" 버튼 클릭
2. 상태: "비활성" 선택
3. "저장" 클릭

【실수로 삭제한 경우】
안타깝지만 복구 방법이 없습니다.

할 수 있는 일:
1. 새로 거래처 생성
2. 동일한 정보 입력
3. 과거 데이터는 복구 불가

예방책:
- 삭제 전 항상 두 번 확인
- 비활성화 우선 고려
- 중요 거래처는 백업 권장


==================================================
4-6. 상품 가격 설정하기
==================================================

거래처별로 상품의 단가를 설정하는 중요한 기능입니다.

【가격 설정 페이지 접근】

방법 1: 거래처 목록에서
1. 거래처 목록 페이지
2. 해당 거래처의 "가격 설정" 버튼 클릭
3. (톱니바퀴 아이콘)
4. 가격 설정 페이지로 이동

방법 2: 직접 주소
- 주소: /admin/clients/[거래처ID]/pricing

【페이지 구성】

헤더:
- 제목: "상품 가격 설정"
- 부제: "[회사명]의 상품별 단가를 설정합니다"

【4가지 상품 카드】
각 상품마다 별도 카드로 표시:

카드 1: 플레이스 유입
- 제목: "플레이스 유입"
- 설명: "네이버 플레이스 유입 접수 서비스"
- 입력 필드:
  * 단가 (포인트): 숫자 입력
  * 거래처에 표시: 스위치 (ON/OFF)

카드 2: 영수증 리뷰
- 제목: "영수증 리뷰"
- 설명: "영수증 기반 리뷰 작성 서비스"
- 입력 필드:
  * 단가 (포인트): 숫자 입력
  * 거래처에 표시: 스위치 (ON/OFF)

카드 3: 카카오맵 리뷰
- 제목: "카카오맵 리뷰"
- 설명: "카카오맵 리뷰 작성 서비스"
- 입력 필드:
  * 단가 (포인트): 숫자 입력
  * 거래처에 표시: 스위치 (ON/OFF)

카드 4: 블로그 배포
- 제목: "블로그 배포"
- 설명: "블로그 콘텐츠 배포 서비스"
- 입력 필드:
  * 단가 (포인트): 숫자 입력
  * 거래처에 표시: 스위치 (ON/OFF)

【하단 버튼 (2개)】
1. 취소: 변경사항 취소하고 거래처 목록으로
2. 저장: 가격 설정 저장 (파란색)

【가격 설정 방법】

1단계: 가격 입력
   - 각 상품의 "단가 (포인트)" 필드에 숫자 입력
   - 0 이상의 정수만 가능
   - 예: 5000, 10000, 15000

2단계: 표시 여부 설정
   - "거래처에 표시" 스위치 조정
   - ON (파란색): 거래처가 해당 상품을 볼 수 있음
   - OFF (회색): 거래처가 해당 상품을 볼 수 없음

3단계: 저장
   - "저장" 버튼 클릭
   - 버튼 텍스트: "저장 중..." 변경
   - 완료 후 거래처 목록으로 자동 이동

【가격 책정 전략】

전략 1: 표준 가격
모든 거래처에 동일한 가격:
- 플레이스 유입: 5,000 P
- 영수증 리뷰: 4,000 P
- 카카오맵 리뷰: 6,000 P
- 블로그 배포: 10,000 P

전략 2: 거래처 규모별 차등
소규모 거래처 (월 10건 미만):
- 플레이스 유입: 6,000 P
- 영수증 리뷰: 5,000 P
- 카카오맵 리뷰: 7,000 P
- 블로그 배포: 12,000 P

대규모 거래처 (월 50건 이상):
- 플레이스 유입: 4,000 P
- 영수증 리뷰: 3,000 P
- 카카오맵 리뷰: 5,000 P
- 블로그 배포: 8,000 P

전략 3: 프로모션 가격
신규 거래처 첫 달:
- 모든 상품 30% 할인
- 예: 플레이스 5,000 P → 3,500 P

전략 4: 선택적 제공
일부 상품만 제공:
- 제공 상품: 표시 ON
- 미제공 상품: 표시 OFF 또는 가격 0

【표시 여부 활용】

상황 1: 모든 상품 제공
- 4가지 모두 표시 ON
- 거래처가 모든 상품 접수 가능

상황 2: 일부 상품만 제공
- 제공하는 상품만 표시 ON
- 나머지는 표시 OFF
- 거래처는 표시된 상품만 보임

상황 3: 준비 중인 상품
- 가격은 입력하되 표시 OFF
- 나중에 표시 ON으로 변경
- 단계적 상품 오픈

상황 4: 일시적 판매 중단
- 표시 OFF로 변경
- 가격 정보는 유지
- 나중에 다시 표시 ON

【실전 예시】

예시 1: 신규 거래처 프리미엄 설정
거래처: 강남 대형 프랜차이즈
계약: 월 100건 이상 보장

설정:
- 플레이스 유입: 3,500 P ✓ 표시
- 영수증 리뷰: 3,000 P ✓ 표시
- 카카오맵 리뷰: 4,500 P ✓ 표시
- 블로그 배포: 7,000 P ✓ 표시

이유: 대량 계약으로 할인 적용

예시 2: 소규모 거래처 표준 설정
거래처: 동네 소규모 카페
계약: 월 5~10건

설정:
- 플레이스 유입: 5,000 P ✓ 표시
- 영수증 리뷰: 4,000 P ✓ 표시
- 카카오맵 리뷰: 6,000 P ✗ 미표시
- 블로그 배포: 10,000 P ✗ 미표시

이유: 기본 상품만 제공

예시 3: 프로모션 가격
거래처: 신규 거래처 (첫 달)
계약: 프로모션 적용

설정:
- 플레이스 유입: 3,500 P ✓ 표시 (30% 할인)
- 영수증 리뷰: 2,800 P ✓ 표시 (30% 할인)
- 카카오맵 리뷰: 4,200 P ✓ 표시 (30% 할인)
- 블로그 배포: 7,000 P ✓ 표시 (30% 할인)

이유: 신규 거래처 유치 프로모션

【주의사항】
⚠️ 가격 설정 안하면 거래처가 접수 불가
⚠️ 표시 OFF면 거래처가 상품을 볼 수 없음
⚠️ 가격은 포인트 단위 (1 P = 1원 개념)
⚠️ 가격 변경은 즉시 적용됨


==================================================
4-7. 포인트 충전/차감
==================================================

거래처의 포인트를 관리하는 핵심 기능입니다.

【포인트 관리 시작】
1. 거래처 목록에서 대상 거래처 찾기
2. "포인트 관리" 버튼 클릭 (달러 아이콘)
3. 포인트 관리 다이얼로그 열림

【다이얼로그 화면】
제목: "포인트 관리"
설명: "[회사명] (현재 포인트: [N] P)"

【탭 구성 (2개)】

탭 1: 충전
- 포인트를 추가하는 탭
- 파란색 강조

탭 2: 차감
- 포인트를 빼는 탭
- 빨간색 강조

【충전 탭 내용】

1. 충전 포인트 (필수)
   - 라벨: "충전 포인트 *"
   - 타입: 숫자 입력
   - 플레이스홀더: "충전할 포인트"
   - 최소값: 1
   - 예: 50000

2. 사유 (선택)
   - 라벨: "사유"
   - 타입: 텍스트 영역 (여러 줄)
   - 플레이스홀더: "충전 사유를 입력하세요"
   - 예: "2024년 11월 계약금 입금"

3. 버튼
   - 취소 (회색)
   - 충전 (파란색)

【차감 탭 내용】

1. 차감 포인트 (필수)
   - 라벨: "차감 포인트 *"
   - 타입: 숫자 입력
   - 플레이스홀더: "차감할 포인트"
   - 최소값: 1
   - 최대값: 현재 보유 포인트
   - 하단 안내: "최대: [N] P"
   - 예: 10000

2. 사유 (선택)
   - 라벨: "사유"
   - 타입: 텍스트 영역
   - 플레이스홀더: "차감 사유를 입력하세요"
   - 예: "과충전 환불"

3. 버튼
   - 취소 (회색)
   - 차감 (빨간색)

【충전 프로세스】

1단계: 충전 탭 선택
   - "충전" 탭 클릭
   - 충전 입력 폼 표시

2단계: 충전 정보 입력
   - 충전 포인트: 금액 입력
   - 사유: 충전 이유 입력 (권장)

3단계: 유효성 확인
   - 포인트는 1 이상이어야 함
   - 숫자만 입력 가능

4단계: 충전 실행
   - "충전" 버튼 클릭
   - 버튼 텍스트: "처리 중..." 변경
   - 처리 대기

5단계: 충전 완료
   - 다이얼로그 닫힘
   - 거래처 목록에서 포인트 업데이트 확인
   - 새 포인트 = 기존 포인트 + 충전 포인트

【차감 프로세스】

1단계: 차감 탭 선택
   - "차감" 탭 클릭
   - 차감 입력 폼 표시

2단계: 차감 정보 입력
   - 차감 포인트: 금액 입력
   - 사유: 차감 이유 입력 (필수 권장)

3단계: 유효성 확인
   - 차감 포인트 ≤ 보유 포인트
   - 초과 시 오류 메시지:
     "차감할 포인트가 보유 포인트보다 많습니다."

4단계: 차감 실행
   - "차감" 버튼 클릭
   - 버튼 텍스트: "처리 중..." 변경

5단계: 차감 완료
   - 다이얼로그 닫힘
   - 포인트 업데이트 확인
   - 새 포인트 = 기존 포인트 - 차감 포인트

【포인트 충전 시나리오】

시나리오 1: 신규 계약 충전
상황: 신규 거래처와 100만원 계약
처리:
1. 포인트 관리 다이얼로그 열기
2. 충전 탭 선택
3. 충전 포인트: 1000000
4. 사유: "2024-11-04 신규 계약 (1,000,000원)"
5. 충전 클릭

시나리오 2: 추가 충전
상황: 기존 거래처가 추가 50만원 입금
처리:
1. 포인트 관리 열기
2. 충전 탭
3. 충전 포인트: 500000
4. 사유: "2024-11-04 추가 충전 (500,000원)"
5. 충전 클릭

시나리오 3: 프로모션 보너스
상황: 신규 가입 이벤트로 10만 포인트 증정
처리:
1. 포인트 관리 열기
2. 충전 탭
3. 충전 포인트: 100000
4. 사유: "신규 가입 이벤트 보너스"
5. 충전 클릭

【포인트 차감 시나리오】

⚠️ 주의: 차감은 특별한 경우에만 사용!

시나리오 1: 과충전 환불
상황: 실수로 100만원 대신 110만원 충전
처리:
1. 포인트 관리 열기
2. 차감 탭
3. 차감 포인트: 100000
4. 사유: "과충전 환불 (10만 포인트)"
5. 차감 클릭

시나리오 2: 계약 위반 패널티
상황: 거래처의 규정 위반으로 패널티
처리:
1. 거래처와 사전 협의 및 동의
2. 포인트 관리 열기
3. 차감 탭
4. 차감 포인트: 50000
5. 사유: "2024-11-04 규정 위반 패널티 (협의 완료)"
6. 차감 클릭

【권장 사항】

충전 시 사유 작성 예시:
- "2024-11-04 신규 계약 (1,000,000원)"
- "2024-11-04 추가 입금 (500,000원)"
- "2024-11-04 환불 처리 (접수 ID: xxx)"
- "2024-11-04 프로모션 보너스"

차감 시 사유 작성 예시:
- "2024-11-04 과충전 환불 (100,000원)"
- "2024-11-04 계약 해지 환불 (200,000원)"
- "2024-11-04 패널티 (규정 위반, 협의 완료)"

【포인트 거래 내역 확인】
충전/차감 후 내역 확인:
1. 좌측 메뉴 "포인트 관리" 클릭
2. 포인트 거래 내역 페이지 열림
3. 방금 처리한 거래 확인
4. 거래 유형, 금액, 사유, 잔액 확인


==================================================
4-8. 비밀번호 초기화
==================================================

거래처가 비밀번호를 잊었을 때 사용하는 기능입니다.

【비밀번호 초기화 시작】
1. 거래처 목록에서 대상 거래처 찾기
2. "비밀번호 재설정" 버튼 클릭 (열쇠 아이콘)
3. 비밀번호 재설정 다이얼로그 열림

【다이얼로그 화면 (초기)】
제목: "비밀번호 재설정"
설명: "[회사명] ([username])의 비밀번호를 재설정합니다."

【옵션 선택】

1. 자동 생성 (기본)
   - 체크박스: "직접 비밀번호 입력" (OFF)
   - 안내 박스:
     "8자리 랜덤 비밀번호가 자동으로 생성됩니다."
   - 특징:
     * 영문 대소문자
     * 숫자
     * 특수문자 조합
     * 예: "Km8$pQ2x"

2. 직접 입력
   - 체크박스: "직접 비밀번호 입력" (ON)
   - 입력 필드 표시:
     * 라벨: "새 비밀번호"
     * 플레이스홀더: "새 비밀번호 입력 (최소 4자)"
     * 최소 길이: 4자
   - 직접 원하는 비밀번호 입력

【하단 버튼 (초기 화면)】
1. 취소 (회색)
2. 비밀번호 재설정 (파란색)

【재설정 프로세스】

1단계: 방식 선택
   - 자동 생성 또는 직접 입력 선택

2단계: 비밀번호 설정 (직접 입력 선택 시)
   - 새 비밀번호 입력
   - 최소 4자 확인

3단계: 재설정 실행
   - "비밀번호 재설정" 버튼 클릭
   - 버튼 텍스트: "재설정 중..." 변경
   - 처리 대기

4단계: 결과 화면 표시
   - 성공 메시지 (초록색 박스):
     "✓ 비밀번호가 성공적으로 재설정되었습니다.
     아래 비밀번호를 거래처에게 전달해주세요."
   
   - 새 비밀번호 표시:
     * 입력 필드 (읽기 전용)
     * 기본값: 가려짐 (●●●●●●●●)
     * 눈 아이콘 버튼: 보기/가리기 전환
     * 복사 버튼: 클립보드에 복사
   
   - 경고 메시지 (노란색 박스):
     "⚠️ 이 비밀번호는 다시 확인할 수 없습니다. 
     반드시 복사하거나 기록해두세요."

5단계: 비밀번호 복사
   - "복사" 버튼 (📋 아이콘) 클릭
   - 알림: "비밀번호가 클립보드에 복사되었습니다."
   - 클립보드에 비밀번호 저장됨

6단계: 거래처에게 전달
   - 안전한 방법으로 전달:
     * 전화로 구두 전달 (권장)
     * 보안 메신저
     * 이메일 (주의!)

7단계: 확인 후 닫기
   - "확인" 버튼 클릭
   - 다이얼로그 닫힘

【비밀번호 보기/가리기】
- 눈 아이콘 버튼 클릭
- 보기 (👁): 비밀번호 표시
  예: "Km8$pQ2x"
- 가리기 (👁‍🗨): 비밀번호 가림
  예: "●●●●●●●●"

【실전 시나리오】

시나리오 1: 거래처가 비밀번호 분실
상황: 거래처로부터 전화
"비밀번호를 잊어버렸어요. 재설정 부탁드립니다."

처리 순서:
1. 본인 확인
   - "회사명과 담당자 성함을 말씀해 주세요"
   - "사업자등록번호 끝 5자리를 말씀해 주세요"

2. 비밀번호 재설정
   - 거래처 목록에서 찾기
   - "비밀번호 재설정" 클릭
   - 자동 생성 선택
   - "비밀번호 재설정" 클릭

3. 새 비밀번호 전달
   - 보기 아이콘 클릭하여 비밀번호 확인
   - 전화로 천천히 불러주기
   - "K-m-8-달러-p-Q-2-x"
   - 거래처가 메모 완료 확인

4. 로그인 테스트
   - "지금 바로 로그인해 보세요"
   - 로그인 성공 확인
   - "로그인 후 비밀번호를 변경하시는 것을 권장드립니다"

5. 다이얼로그 닫기
   - "확인" 버튼 클릭

시나리오 2: 정기 보안 비밀번호 변경
상황: 보안 정책상 3개월마다 비밀번호 변경

처리 순서:
1. 대상 거래처 목록 작성
   - 3개월 이상 비밀번호 미변경 거래처

2. 거래처에게 사전 공지
   - 이메일/문자: "보안을 위해 비밀번호를 변경합니다"
   - 변경 일정 안내

3. 일괄 재설정
   - 각 거래처마다:
     * "비밀번호 재설정" 클릭
     * 자동 생성 선택
     * 재설정 실행
     * 새 비밀번호 복사
     * 엑셀/메모장에 기록
   
4. 거래처에게 전달
   - 각 거래처에게 개별 전달
   - 보안 메신저 또는 전화

시나리오 3: 직접 입력으로 간단한 비밀번호
상황: 고령의 거래처 담당자, 복잡한 비밀번호 어려움

처리 순서:
1. "비밀번호 재설정" 클릭
2. "직접 비밀번호 입력" 체크
3. 간단한 비밀번호 입력
   - 예: "cafe1234"
   - 최소 4자 이상
4. "비밀번호 재설정" 클릭
5. 새 비밀번호 전달

【주의사항】
⚠️ 비밀번호는 재확인 불가 - 반드시 복사/기록!
⚠️ 안전한 채널로만 전달 (전화 권장)
⚠️ 본인 확인 후 재설정
⚠️ 재설정 후 즉시 거래처에게 알리기
⚠️ 공개 채널(단체 문자, 공개 게시판) 사용 금지

【보안 모범 사례】
✓ 전화로 구두 전달 (가장 안전)
✓ 보안 메신저 (카카오톡, 텔레그램)
✓ 일대일 이메일 (비밀번호는 별도 전화)
✓ 로그인 테스트 확인
✓ 비밀번호 변경 권장

==================================================
5-1. 전체 접수 내역 조회
==================================================

전체 접수 내역 페이지는 모든 거래처의 상품 접수를 한곳에서 관리합니다.

【페이지 접근】
- 좌측 메뉴 "접수 내역" 클릭
- 주소: /admin/submissions

【페이지 구성】

헤더:
- 제목: "전체 접수 내역"
- 부제: "모든 거래처의 접수 내역을 조회하고 관리합니다"

카드 헤더:
- 제목: "전체 접수 내역 (N / M건)"
  * N: 현재 필터링된 건수
  * M: 전체 건수
- 우측: "엑셀 다운로드" 버튼

【필터 영역 (5개)】

1. 검색
   - 플레이스홀더: "거래처명, 업체명..."
   - 기능: 실시간 검색
   - 검색 대상:
     * 거래처 회사명
     * 접수한 업체명

2. 상품 유형
   - 드롭다운 선택
   - 옵션:
     * 전체 (기본값)
     * 플레이스 유입
     * 영수증 리뷰
     * 카카오맵 리뷰
     * 블로그 배포

3. 상태
   - 드롭다운 선택
   - 옵션:
     * 전체 (기본값)
     * 대기중
     * 진행중
     * 완료
     * 취소

4. 기간
   - 드롭다운 선택
   - 옵션:
     * 전체 (기본값)
     * 오늘
     * 최근 7일
     * 최근 30일

5. 정렬
   - 드롭다운 선택
   - 옵션:
     * 최신순 (기본값)
     * 오래된순
     * 포인트 높은순
     * 포인트 낮은순

【모바일 화면】
접수가 카드 형태로 표시:
- 각 카드에 접수 정보 요약
- 상품 유형 배지
- 상태 드롭다운 (변경 가능)
- 상세보기 버튼
- 스크롤하여 탐색

【데스크탑 화면】
테이블 형태로 표시:

테이블 컬럼 (8개):
1. 접수일시: YYYY-MM-DD HH:MM 형식
2. 거래처: 거래처 회사명
3. 상품유형: 배지 형태
4. 업체명: 접수한 업체 이름
5. 상세내용: 접수 세부사항 요약
6. 사용 포인트: 포인트 (P 표시)
7. 상태: 드롭다운 (변경 가능)
8. 상세보기: 눈 아이콘 버튼

【접수 상세내용 표시】
상품별로 다른 형식:

플레이스 유입:
- "일 N타 × M일"
- 예: "일 100타 × 5일"

영수증 리뷰:
- "총 N타"
- 예: "총 50타"

카카오맵 리뷰:
- "총 N타"
- 예: "총 30타"

블로그 배포:
- "유형 / 일 N타 × M일"
- 예: "리뷰어형 / 일 2타 × 10일"


==================================================
5-2. 필터링 및 검색
==================================================

대량의 접수 내역에서 원하는 정보를 빠르게 찾는 방법입니다.

【검색 기능】

1단계: 검색어 입력
   - 검색 입력란 클릭
   - 거래처명 또는 업체명 입력

2단계: 실시간 필터링
   - 타이핑과 동시에 결과 표시
   - 매칭되는 접수만 표시

예시:
- "강남" 입력 → "강남 커피", "서울 강남점" 모두 검색
- "스타벅스" 입력 → 스타벅스 관련 모든 접수

【상품 유형 필터】

활용 예시:
- 플레이스 유입만: 특정 상품 성과 분석
- 블로그 배포만: 고단가 상품 모니터링

【상태 필터】

상태별 활용:

대기중:
- 처리가 필요한 접수
- 매일 아침 확인 권장

진행중:
- 현재 작업 중인 접수
- 진행 상황 모니터링

완료:
- 완료된 접수 확인
- 성과 보고서 작성 시

취소:
- 취소된 접수 확인
- 취소 사유 분석

【기간 필터】

오늘:
- 오늘 접수된 건만
- 일일 현황 파악

최근 7일:
- 주간 현황
- 주간 보고서 작성

최근 30일:
- 월간 현황
- 월간 보고서 작성

전체:
- 모든 기간
- 전체 통계 분석

【정렬 기능】

최신순 (기본):
- 최근 접수부터 표시
- 일반적인 모니터링

오래된순:
- 오래된 접수부터
- 오래 대기 중인 접수 확인

포인트 높은순:
- 고액 접수 우선
- 중요 건 우선 처리

포인트 낮은순:
- 소액 접수 우선
- 빠른 처리 가능 건

【필터 조합 활용】

예시 1: 오늘 대기중인 접수
1. 기간: "오늘" 선택
2. 상태: "대기중" 선택
→ 오늘 처리해야 할 접수 확인

예시 2: 특정 거래처의 블로그 접수
1. 검색: "강남 커피" 입력
2. 상품 유형: "블로그 배포" 선택
→ 해당 거래처의 블로그 접수만 표시

예시 3: 이번 주 완료된 고액 접수
1. 기간: "최근 7일" 선택
2. 상태: "완료" 선택
3. 정렬: "포인트 높은순" 선택
→ 이번 주 주요 완료 건 확인


==================================================
5-3. 접수 상태 변경
==================================================

접수의 진행 상태를 관리하는 중요한 기능입니다.

【상태 변경 방법】

데스크탑:
1. 테이블에서 해당 접수 찾기
2. "상태" 컬럼의 드롭다운 클릭
3. 새로운 상태 선택
4. 즉시 적용

모바일:
1. 접수 카드 찾기
2. 상태 배지 영역의 드롭다운 터치
3. 새로운 상태 선택
4. 즉시 적용

【상태 종류 (4가지)】

1. 대기중 (회색 테두리 배지)
   - 의미: 아직 처리 시작 안함
   - 언제 사용: 신규 접수 초기 상태
   - 다음 단계: → 진행중

2. 진행중 (파란색 배지)
   - 의미: 현재 작업 중
   - 언제 사용: 처리 시작 후
   - 다음 단계: → 완료 또는 취소

3. 완료 (초록색 배지)
   - 의미: 작업 완료
   - 언제 사용: 모든 작업 끝난 후
   - 최종 상태

4. 취소 (빨간색 배지)
   - 의미: 접수 취소
   - 언제 사용: 오류, 거래처 요청 등
   - 최종 상태

【상태 변경 프로세스】

표준 프로세스:
대기중 → 진행중 → 완료

변경 시점:

대기중 → 진행중:
- 작업을 시작할 때
- 예: 작업자에게 할당 완료

진행중 → 완료:
- 모든 작업이 끝났을 때
- 예: 리포트 전달 완료

진행중 → 취소:
- 문제 발생으로 진행 불가
- 예: 거래처 요청, 정보 오류

대기중 → 취소:
- 처리 시작 전 취소
- 예: 거래처 즉시 취소 요청

【상태 변경 시나리오】

시나리오 1: 정상 처리
1. 신규 접수 → 자동으로 "대기중"
2. 작업 시작 → "진행중"으로 변경
3. 작업 완료 → "완료"로 변경

시나리오 2: 처리 중 취소
1. 접수 상태: "진행중"
2. 거래처가 취소 요청
3. "취소"로 변경
4. 포인트 환불 처리

시나리오 3: 즉시 취소
1. 접수 상태: "대기중"
2. 정보 오류 발견
3. "취소"로 변경
4. 포인트 환불 처리

【일괄 상태 변경】

여러 건을 동시에 변경해야 할 때:

1. 필터로 대상 선별
   예: 어제 대기중인 접수 모두

2. 각 접수마다 개별 변경
   - 드롭다운에서 "진행중" 선택
   - 다음 접수로 이동
   - 반복

3. 변경 확인
   - 필터 재적용
   - 상태 변경 확인


==================================================
5-4. 접수 상세보기
==================================================

접수의 모든 정보를 확인하는 기능입니다.

【상세보기 열기】

데스크탑:
1. 테이블에서 해당 접수 찾기
2. "상세보기" 컬럼의 눈 아이콘 버튼 클릭
3. 상세보기 다이얼로그 열림

모바일:
1. 접수 카드 찾기
2. "상세보기" 버튼 터치
3. 상세보기 다이얼로그 열림

【다이얼로그 내용】

제목: "접수 상세 정보"
부제: "접수의 상세 내용을 확인하세요"

【표시 정보】

기본 정보 섹션:
- 거래처: 회사명
- 접수일시: YYYY-MM-DD HH:MM
- 상품 유형: 배지
- 상태: 배지

접수 상세 정보:
- 업체명
- 포인트 사용액
- 상세 내용 (상품별 상이)

상품별 추가 정보:

플레이스 유입:
- 플레이스 URL
- 일일 타수
- 총 일수
- 시작일
- 비고

영수증 리뷰:
- 플레이스 URL
- 총 타수
- 사진 첨부 여부
- 대본 첨부 여부
- 가이드 텍스트
- 첨부 파일들

카카오맵 리뷰:
- 카카오맵 URL
- 총 타수
- 텍스트/사진 리뷰 비율
- 첨부 파일들

블로그 배포:
- 배포 유형 (리뷰어형/영상형/자동화형)
- 콘텐츠 유형
- 플레이스 URL
- 일일/총 타수
- 키워드 목록
- 가이드 텍스트
- 첨부 파일들

【하단 버튼】
- 닫기: 다이얼로그 닫기

【활용 방법】

확인 사항:
1. 접수 정보 정확성
   - URL 유효성
   - 타수 적정성
   - 첨부 파일 완전성

2. 작업 요구사항
   - 가이드 텍스트 확인
   - 특별 요청 사항
   - 주의사항

3. 거래처 연락
   - 정보 미비 시 연락
   - 추가 자료 요청
   - 진행 상황 공유


==================================================
5-5. 엑셀 다운로드
==================================================

접수 내역을 엑셀 파일로 다운로드하여 분석할 수 있습니다.

【다운로드 방법】

1단계: 필터 설정 (선택사항)
   - 원하는 기간 선택
   - 상품 유형 선택
   - 상태 선택

2단계: 다운로드 버튼 클릭
   - 카드 헤더 우측 상단
   - "엑셀 다운로드" 버튼 클릭

3단계: 파일 다운로드
   - 파일명: "접수내역_YYYY-MM-DD.xlsx"
   - 다운로드 폴더에 자동 저장

【엑셀 파일 구성】

컬럼 (7개):
1. 접수일시
2. 거래처
3. 상품유형
4. 업체명
5. 상세내용
6. 사용포인트
7. 상태

【활용 예시】

용도 1: 월간 보고서
1. 기간: "최근 30일" 선택
2. 엑셀 다운로드
3. 피벗 테이블로 분석:
   - 상품별 집계
   - 거래처별 집계
   - 일별 추이

용도 2: 거래처별 정산
1. 검색: 특정 거래처 입력
2. 기간: 해당 월 선택
3. 엑셀 다운로드
4. 정산서 작성 자료로 활용

용도 3: 상품 성과 분석
1. 상품 유형: 특정 상품 선택
2. 기간: 분석 기간 선택
3. 엑셀 다운로드
4. 완료율, 평균 포인트 계산

제6장: 포인트 관리

==================================================
6-1. 포인트 거래 내역 조회
==================================================

모든 포인트 거래를 조회하는 페이지입니다.

【페이지 접근】
- 좌측 메뉴 "포인트 관리" 클릭
- 주소: /admin/points

【페이지 구성】

헤더:
- 제목: "포인트 관리"
- 부제: "전체 거래처의 포인트 거래 내역을 조회합니다"

카드 헤더:
- 제목: "전체 포인트 거래 내역 (N / M건)"

【필터 영역 (4개)】

1. 거래처 검색
   - 플레이스홀더: "거래처명..."
   - 실시간 검색

2. 거래 유형
   - 드롭다운
   - 옵션:
     * 전체
     * 충전
     * 차감

3. 기간
   - 드롭다운
   - 옵션:
     * 전체
     * 오늘
     * 최근 7일
     * 최근 30일

4. 정렬
   - 드롭다운
   - 옵션:
     * 최신순
     * 오래된순
     * 금액 높은순
     * 금액 낮은순

【모바일 화면】
거래 내역이 카드 형태로 표시:
- 거래처명
- 거래 일시
- 거래 유형 배지 (충전/차감)
- 거래 금액
- 거래 후 잔액
- 설명

【데스크탑 화면】
테이블 형태로 표시:

테이블 컬럼 (6개):
1. 거래 일시: YYYY-MM-DD HH:MM
2. 거래처: 회사명
3. 유형: 배지 (충전 = 파란색, 차감 = 빨간색)
4. 금액: +/- 표시 + 천단위 콤마
5. 거래 후 잔액: 천단위 콤마 + "원"
6. 설명: 거래 사유

【거래 유형 표시】

충전:
- 배지 색상: 파란색
- 금액 표시: "+N 원"
- 의미: 포인트 증가

차감:
- 배지 색상: 빨간색
- 금액 표시: "-N 원"
- 의미: 포인트 감소


==================================================
6-2. 거래 내역 필터링
==================================================

【검색 활용】

특정 거래처의 거래만 보기:
1. 검색란에 거래처명 입력
2. 해당 거래처의 모든 거래 표시

【거래 유형 필터】

충전만 보기:
1. 거래 유형: "충전" 선택
2. 포인트 충전 내역만 표시
3. 활용: 입금 내역 확인

차감만 보기:
1. 거래 유형: "차감" 선택
2. 포인트 차감 내역만 표시
3. 활용: 환불 내역 확인

【기간 필터】

오늘 거래:
- 오늘 발생한 거래만
- 일일 정산

최근 7일:
- 주간 거래 확인
- 주간 보고서

최근 30일:
- 월간 거래 확인
- 월간 정산

【정렬 활용】

최신순:
- 최근 거래부터 표시
- 일반적인 모니터링

금액 높은순:
- 고액 거래 확인
- 중요 거래 체크

【실전 활용 예시】

예시 1: 특정 거래처 월간 정산
1. 검색: "강남 커피" 입력
2. 기간: "최근 30일" 선택
3. 결과 확인 및 정산

예시 2: 오늘 충전 내역
1. 거래 유형: "충전" 선택
2. 기간: "오늘" 선택
3. 입금 확인

예시 3: 고액 거래 모니터링
1. 정렬: "금액 높은순" 선택
2. 고액 거래 확인
3. 이상 거래 체크

제7장: AS 요청 관리
============================================================================

==================================================
7-1. AS 요청 목록 조회
==================================================

거래처의 AS 요청을 확인하고 처리하는 페이지입니다.

【페이지 접근】
- 좌측 메뉴 "AS 관리" 클릭
- 주소: /admin/as-requests

【페이지 구성】

헤더:
- 제목: "AS 요청 관리"
- 부제: "거래처의 AS 요청을 확인하고 처리합니다"

카드 헤더:
- 제목: "전체 AS 요청 (N건)"

【AS 요청 정보】

모바일 카드:
- 거래처명
- 요청일시
- 상품 유형 배지
- 예정/실제 수량
- 미달률 (빨간색 강조)
- 상태 배지
- 상세보기 버튼
- 처리 버튼

데스크탑 테이블:

컬럼 (7개):
1. 요청일시
2. 거래처
3. 상품유형
4. 예정/실제
5. 미달률
6. 상태
7. 처리 버튼들

【상태 종류】

대기중 (파란색 배지):
- 아직 처리 안됨
- 즉시 확인 필요

승인 (초록색 배지):
- AS 승인됨
- 처리 완료

거절 (빨간색 배지):
- AS 거절됨
- 처리 완료


==================================================
7-2. AS 요청 상세보기
==================================================

AS 요청의 자세한 내용을 확인합니다.

【상세보기 열기】
1. 목록에서 "상세보기" 버튼 클릭
2. 상세보기 다이얼로그 열림

【다이얼로그 내용】

제목: "AS 요청 상세 정보"
부제: "AS 요청의 상세 내용을 확인하세요"

【기본 정보】
- 거래처
- 요청일시
- 상품 유형 배지
- 상태 배지

【미달 현황 박스】
회색 배경 박스에 표시:
- 예정 수량: N개
- 실제 수량: M개
- 미달률: X% (빨간색 강조)
- 미달 수량: (N-M)개

【AS 신청 사유】
회색 배경 박스에 표시:
- 거래처가 작성한 AS 신청 사유
- 여러 줄 표시 가능

【관리자 응답】 (있는 경우)
회색 배경 박스에 표시:
- 관리자가 작성한 응답 내용
- 승인/거절 사유

【하단 버튼】

대기중 상태:
- 닫기 버튼
- 처리하기 버튼 (파란색)

승인/거절 상태:
- 닫기 버튼만


==================================================
7-3. AS 요청 처리하기
==================================================

AS 요청을 승인 또는 거절하는 프로세스입니다.

【처리 시작】
1. 목록 또는 상세보기에서 "처리" 버튼 클릭
2. AS 요청 처리 다이얼로그 열림

【다이얼로그 내용】

제목: "AS 요청 처리"
부제: "AS 요청을 검토하고 승인 또는 거절하세요"

【요청 정보 확인】

1. 거래처
   - 거래처명 표시

2. 상품 유형
   - 배지로 표시

3. 미달 내역
   - "예정: N / 실제: M (미달률: X%)"

4. 요청 사유
   - 거래처가 작성한 내용

【처리 입력 필드】

1. 처리 상태 (필수)
   - 드롭다운 선택
   - 옵션:
     * 승인
     * 거절

2. 관리자 응답 (선택)
   - 텍스트 영역 (여러 줄)
   - 플레이스홀더: "AS 처리 내용을 입력하세요"
   - 5줄 높이

【하단 버튼】
- 취소
- 처리 완료 (파란색)

【처리 프로세스】

1단계: 요청 내용 검토
   - 미달률 확인
   - 요청 사유 검토
   - 정당성 판단

2단계: 처리 상태 선택
   - 승인 또는 거절 선택

3단계: 응답 작성
   - 처리 이유 작성
   - 향후 조치 안내

4단계: 처리 완료
   - "처리 완료" 버튼 클릭
   - 버튼 텍스트: "처리 중..." 변경

5단계: 결과 확인
   - 다이얼로그 닫힘
   - 목록 업데이트
   - 상태 변경 확인

【AS 처리 시나리오】

시나리오 1: 정당한 AS 승인
상황: 
- 예정: 100타
- 실제: 75타
- 미달률: 25%
- 사유: "작업 미완료 타 발생"

처리:
1. 처리 상태: "승인" 선택
2. 관리자 응답:
   "확인했습니다. 미달 25타에 대해 
   포인트 환불 처리하겠습니다.
   
   환불 포인트: 12,500 P
   환불 완료 후 알림 드리겠습니다."
3. 처리 완료 클릭
4. 포인트 환불 처리 (거래처 관리 → 포인트 관리)

시나리오 2: 정당하지 않은 AS 거절
상황:
- 예정: 100타
- 실제: 95타
- 미달률: 5%
- 사유: "5타 미달 환불 요청"

처리:
1. 처리 상태: "거절" 선택
2. 관리자 응답:
   "미달률이 20% 미만으로 
   AS 기준에 해당하지 않습니다.
   
   AS는 미달률 20% 이상일 때만 
   가능합니다.
   
   양해 부탁드립니다."
3. 처리 완료 클릭

시나리오 3: 부분 승인
상황:
- 예정: 100타
- 실제: 70타
- 미달률: 30%
- 사유: "30타 미달 발생"

처리:
1. 처리 상태: "승인" 선택
2. 관리자 응답:
   "확인했습니다.
   
   검토 결과 실제 미달은 25타로 
   확인되었습니다.
   
   25타에 대한 포인트 환불 처리:
   환불 포인트: 12,500 P
   
   처리 완료 후 연락드리겠습니다."
3. 처리 완료 클릭
4. 해당 포인트 환불

【AS 처리 후 조치】

승인 시:
1. 포인트 환불 처리
   - 거래처 관리 → 포인트 관리
   - 충전 탭에서 환불 포인트 입력
   - 사유: "AS 승인 - 접수 ID: xxx"

2. 거래처에게 연락
   - 전화 또는 이메일
   - 환불 완료 안내
   - 사과 및 재발 방지 약속

3. 내부 기록
   - AS 발생 원인 분석
   - 재발 방지 대책 수립

거절 시:
1. 거래처에게 연락
   - 거절 사유 상세 설명
   - AS 기준 안내
   - 추가 문의 응대

2. 분쟁 대비
   - 거절 사유 문서화
   - 증빙 자료 보관

제8장: 리포트 관리
============================================================================

==================================================
8-1. 리포트 업로드하기
==================================================

완료된 접수 건에 대한 결과 리포트를 업로드합니다.

⚠️ 주의: 현재 리포트 관리 기능은 기본 제공되지 않습니다.
필요 시 관리자에게 문의하여 기능을 활성화하세요.

【페이지 접근】
- 좌측 메뉴 (리포트 관리 메뉴가 있는 경우)
- 주소: /admin/reports

【리포트 업로드 폼】

제목: "리포트 업로드"
설명: "접수 건에 대한 작업 결과 리포트를 업로드합니다"

【입력 필드 (3개)】

1. 상품 유형 (필수)
   - 드롭다운 선택
   - 옵션:
     * 플레이스 유입
     * 영수증 리뷰
     * 카카오맵 리뷰
     * 블로그 배포

2. 접수 ID (필수)
   - 텍스트 입력
   - 플레이스홀더: "접수 ID를 입력하세요"
   - 안내: "접수 내역 페이지에서 해당 건의 ID를 확인하세요"

3. 리포트 파일 (필수)
   - 파일 선택
   - 허용 형식: PDF, Excel, Word, ZIP
   - 최대 크기: 50MB
   - 선택 후 파일명과 크기 표시

【업로드 버튼】
- 아이콘: 업로드
- 텍스트: "리포트 업로드"
- 색상: 파란색

【업로드 프로세스】

1단계: 접수 ID 확인
   - 접수 내역 페이지에서 해당 접수 찾기
   - ID 복사

2단계: 리포트 준비
   - 작업 결과를 파일로 정리
   - PDF, Excel, Word 중 선택
   - 여러 파일은 ZIP으로 압축

3단계: 정보 입력
   - 상품 유형 선택
   - 접수 ID 붙여넣기
   - 파일 선택

4단계: 업로드 실행
   - "리포트 업로드" 버튼 클릭
   - 버튼 텍스트: "업로드 중..." 변경
   - 진행률 표시 (큰 파일)

5단계: 완료 확인
   - 성공 메시지 (초록색)
     "리포트가 성공적으로 업로드되었습니다."
   - 폼 초기화

【리포트 작성 가이드】

플레이스 유입 리포트:
- 일별 유입 수치
- 유입 소스 분석
- 스크린샷

영수증 리뷰 리포트:
- 리뷰 작성 완료 목록
- 리뷰 URL 또는 스크린샷
- 작성 일시

카카오맵 리뷰 리포트:
- 리뷰 작성 완료 목록
- 리뷰 URL 또는 스크린샷
- 평점 및 내용 요약

블로그 배포 리포트:
- 배포된 블로그 URL 목록
- 일별 배포 현황
- 주요 키워드 순위

【오류 처리】

파일 크기 초과:
"파일 크기는 50MB 이하여야 합니다."
→ 파일 압축 또는 분할

형식 오류:
"지원하지 않는 파일 형식입니다."
→ PDF, Excel, Word, ZIP 중 선택

접수 ID 오류:
"존재하지 않는 접수 ID입니다."
→ 접수 내역에서 ID 재확인

9-3. 거래처 문의 대응
==================================================

거래처로부터 전화/이메일 문의가 왔을 때 대응 방법입니다.

【유형 1: 로그인 문제】

문의: "로그인이 안돼요"

대응 순서:
1. 상황 파악:
   "어떤 메시지가 표시되나요?"
   - 아이디/비밀번호 오류
   - 페이지가 안 열림
   - 기타

2. 아이디/비밀번호 오류 시:
   "아이디를 다시 한 번 말씀해 주시겠어요?"
   → 거래처 관리에서 확인
   → 맞으면 비밀번호 재설정
   → 틀리면 정확한 아이디 알려주기

3. 페이지 안 열림:
   "어떤 브라우저를 사용하시나요?"
   "주소를 정확히 입력하셨나요?"
   → 올바른 주소 안내
   → 브라우저 변경 권장

4. 임시 조치:
   "임시로 새 비밀번호를 발급해 드리겠습니다"
   → 비밀번호 초기화 진행

【유형 2: 포인트 문의】

문의: "포인트가 이상해요"

대응 순서:
1. 현재 포인트 확인:
   "현재 보유 포인트가 얼마로 표시되나요?"
   → 거래처 관리에서 실제 확인

2. 불일치 시:
   "새로고침(F5)을 한 번 눌러주시겠어요?"
   → 캐시 문제일 수 있음

3. 여전히 불일치:
   → 포인트 거래 내역 확인
   → 최근 거래 설명
   → 오류 발견 시 즉시 수정

예시:
"확인해보니 어제 50,000 포인트가 
차감되었습니다.
플레이스 유입 10건을 신청하셔서
자동 차감된 것입니다.
접수 내역에서 확인하실 수 있습니다."

【유형 3: 접수 오류】

문의: "접수가 안돼요"

대응 순서:
1. 오류 상황 파악:
   "어떤 메시지가 나타나나요?"
   - 포인트 부족
   - 입력 오류
   - 기타

2. 포인트 부족:
   → 현재 포인트 확인
   → 필요 포인트 안내
   → 충전 필요 시 입금 안내

3. 입력 오류:
   "어떤 항목에서 오류가 나나요?"
   → 해당 항목 입력 방법 안내
   → 예시 제공

4. 시스템 오류:
   → 접수 내역 확인
   → 실제로 접수되었는지 확인
   → 중복 접수 방지 안내

【유형 4: 작업 문의】

문의: "작업은 언제 완료되나요?"

대응 순서:
1. 접수 상태 확인:
   → 접수 내역 페이지에서 검색
   → 현재 상태 확인

2. 대기중:
   "현재 대기중 상태입니다.
   곧 작업을 시작하겠습니다.
   예상 완료일: [날짜]"

3. 진행중:
   "현재 작업 진행 중입니다.
   예상 완료일: [날짜]
   완료 즉시 연락드리겠습니다."

4. 완료:
   "작업이 완료되었습니다.
   리포트를 확인해 주세요.
   [리포트 위치 안내]"

【유형 5: 환불 요청】

문의: "취소하고 환불받고 싶어요"

대응 순서:
1. 환불 사유 확인:
   "어떤 이유로 환불을 원하시나요?"

2. 접수 상태 확인:
   → 접수 내역에서 해당 건 찾기

3. 환불 가능 판단:
   - 대기중: 즉시 환불 가능
   - 진행중: 진행 정도 확인 후 부분 환불
   - 완료: 환불 불가, 설명 필요

4. 환불 처리:
   → 접수 상태를 "취소"로 변경
   → 포인트 충전으로 환불
   → 사유: "접수 취소 환불 - ID: xxx"

5. 확인:
   "환불이 완료되었습니다.
   포인트를 확인해 주세요."

9-4. 대량 접수 처리 방법
==================================================

한 번에 많은 접수가 들어왔을 때 효율적으로 처리하는 방법입니다.

【상황】
월요일 아침, 주말 동안 50건의 접수 누적

【처리 전략】

1단계: 우선순위 분류 (10분)

1-1. 접수 내역 페이지 이동

1-2. 필터 설정:
   - 상태: "대기중" 선택
   - 정렬: "포인트 높은순" 선택

1-3. 우선순위 결정:
   높음: 고액 접수 (50,000 P 이상)
   중간: 중액 접수 (20,000~50,000 P)
   낮음: 소액 접수 (20,000 P 미만)

2단계: 일괄 처리 준비 (5분)

2-1. 상품별로 그룹화:
   - 상품 유형 필터 사용
   - 플레이스 유입: N건
   - 영수증 리뷰: N건
   - 카카오맵 리뷰: N건
   - 블로그 배포: N건

2-2. 작업자 배정 계획:
   - 작업자 A: 플레이스 유입
   - 작업자 B: 영수증/카카오맵
   - 작업자 C: 블로그

3단계: 상태 일괄 변경 (15분)

3-1. 각 상품별로:
   - 해당 상품만 필터링
   - 대기중 → 진행중으로 하나씩 변경
   - 드롭다운에서 "진행중" 선택

3-2. 변경하면서 메모:
   - 특이사항 있는 접수 표시
   - 추가 확인 필요 건 표시

4단계: 작업 지시 (10분)

4-1. 작업자별 리스트 전달:
   → 엑셀 다운로드
   → 작업자별로 필터링
   → 각 작업자에게 이메일/메신저 전달

4-2. 특이사항 공유:
   "A 거래처의 접수는 
   추가 파일 확인 필요합니다."

5단계: 진행 모니터링 (지속)

5-1. 주기적 확인:
   - 오전/오후 각 1회
   - 진행중 → 완료 건수 확인

5-2. 문제 발생 시:
   - 즉시 거래처 연락
   - 해결 방안 협의

6단계: 완료 처리 (지속)

6-1. 완료 보고 받으면:
   - 상태를 "완료"로 변경
   - 리포트 업로드 (있는 경우)

6-2. 거래처 통보:
   - 이메일/문자로 완료 안내
   - 리포트 전달 (있는 경우)

【효율화 팁】

1. 단축키 활용:
   - Ctrl + F: 페이지 내 검색
   - F5: 새로고침
   - Ctrl + Click: 새 탭으로 열기

2. 다중 탭 활용:
   - 탭 1: 접수 내역 (대기중)
   - 탭 2: 접수 내역 (진행중)
   - 탭 3: 거래처 관리

3. 엑셀 활용:
   - 다운로드 후 가공
   - 작업자별로 시트 분리
   - 완료 체크리스트 작성

4. 커뮤니케이션:
   - 단체 메신저방 활용
   - 실시간 진행 상황 공유
   - 문제 발생 즉시 공유

10-1. 자주 발생하는 문제
==================================================

【문제 1: 거래처가 로그인 못함】

증상:
- 거래처가 "아이디 또는 비밀번호가 틀립니다" 메시지

원인 및 해결:
1. 아이디 오타
   → 정확한 아이디 알려주기
   
2. 비밀번호 분실
   → 비밀번호 초기화 진행

3. 계정 비활성화
   → 거래처 관리에서 상태 확인
   → 활성으로 변경

【문제 2: 포인트가 이상함】

증상:
- 거래처가 "포인트가 맞지 않아요"

원인 및 해결:
1. 자동 차감 모름
   → 접수 내역 설명
   → 각 접수의 포인트 설명

2. 캐시 문제
   → 새로고침 안내
   → 브라우저 캐시 삭제

3. 실제 오류
   → 포인트 거래 내역 확인
   → 오류 발견 시 즉시 수정
   → 관리자에게 보고

【문제 3: 접수가 안됨】

증상:
- 거래처가 "접수 버튼을 눌러도 안돼요"

원인 및 해결:
1. 포인트 부족
   → 현재 포인트 확인
   → 필요 포인트 안내
   → 충전 안내

2. 가격 미설정
   → 가격 설정 페이지에서 확인
   → 가격 및 표시 설정

3. 입력 오류
   → 필수 항목 확인
   → 올바른 입력 형식 안내

【문제 4: 상태 변경 안됨】

증상:
- 관리자가 상태 변경 시도 시 오류

원인 및 해결:
1. 네트워크 오류
   → 페이지 새로고침
   → 재시도

2. 권한 문제
   → 로그아웃 후 재로그인
   → 관리자 계정 확인

3. 시스템 오류
   → 개발자에게 보고

【문제 5: 엑셀 다운로드 안됨】

증상:
- "엑셀 다운로드" 버튼 클릭 시 반응 없음

원인 및 해결:
1. 데이터 없음
   → 필터 확인
   → 실제 데이터 존재 확인

2. 브라우저 팝업 차단
   → 팝업 허용 설정
   → 다운로드 다시 시도

3. 파일 용량 과다
   → 기간 필터로 범위 축소
   → 분할 다운로드

10-2. 자주 묻는 질문
==================================================

Q1: 거래처 아이디를 변경할 수 있나요?
A: 아니요, 아이디는 생성 후 변경할 수 없습니다.
   새 아이디가 필요하면 새로운 거래처를 생성해야 합니다.
   기존 거래처는 비활성화하세요.

Q2: 거래처별로 다른 가격을 설정할 수 있나요?
A: 네, 가능합니다.
   거래처 관리 → 해당 거래처 → 가격 설정
   각 거래처마다 개별적으로 설정할 수 있습니다.

Q3: 모든 거래처에게 일괄로 포인트를 충전할 수 있나요?
A: 아니요, 일괄 충전 기능은 없습니다.
   각 거래처마다 개별적으로 충전해야 합니다.

Q4: 접수를 삭제할 수 있나요?
A: 아니요, 접수는 삭제할 수 없습니다.
   대신 상태를 "취소"로 변경하세요.
   포인트 환불은 별도로 처리해야 합니다.

Q5: 거래처가 비밀번호를 스스로 변경할 수 있나요?
A: 현재는 관리자만 비밀번호를 변경할 수 있습니다.
   거래처가 비밀번호 변경을 원하면
   관리자가 재설정해 주어야 합니다.

Q6: AS 승인 시 포인트가 자동 환불되나요?
A: 아니요, 자동 환불되지 않습니다.
   AS 승인 후 별도로 포인트 관리에서
   충전으로 환불 처리해야 합니다.

Q7: 리포트는 어디에 저장되나요?
A: 리포트는 서버에 저장됩니다.
   현재는 별도 조회 기능이 없으므로
   업로드 시 로컬에도 백업하세요.

Q8: 거래처가 자기 접수 내역을 볼 수 있나요?
A: 네, 거래처는 자신의 접수 내역을
   거래처 화면에서 확인할 수 있습니다.
   다른 거래처의 내역은 볼 수 없습니다.

Q9: 통계 데이터는 실시간인가요?
A: 네, 모든 통계는 실시간으로 업데이트됩니다.
   페이지를 새로고침하면 최신 데이터를 볼 수 있습니다.

Q10: 과거 데이터를 백업하려면?
A: 정기적으로 엑셀 다운로드를 권장합니다.
   - 접수 내역: 월 1회 전체 다운로드
   - 포인트 내역: 화면 캡처 또는 수동 기록
   별도 서버 백업은 개발자에게 문의하세요.

Q11: 모바일에서도 모든 기능을 사용할 수 있나요?
A: 네, 모든 관리자 기능은 모바일에서도 사용 가능합니다.
   화면 크기에 따라 레이아웃만 다르게 표시됩니다.

Q12: 여러 관리자가 동시에 사용해도 되나요?
A: 네, 여러 관리자가 동시 접속 가능합니다.
   단, 같은 관리자 계정을 공유하면
   누가 어떤 작업을 했는지 추적이 어렵습니다.
   관리자별로 별도 계정 사용을 권장합니다.

D-1. 시스템 접속 불가
==================================================

【증상】
관리자 또는 모든 사용자가 시스템에 접속할 수 없음

【즉시 조치】
1. 본인 네트워크 확인
   - 다른 웹사이트 접속 테스트
   - WiFi/인터넷 연결 확인

2. 다른 기기에서 시도
   - 스마트폰으로 접속 시도
   - 다른 컴퓨터에서 시도

3. 시스템 상태 확인
   - 서버 상태 확인 (호스팅 업체)
   - 개발자에게 즉시 연락

【거래처 공지】
- 단체 문자/이메일:
  "시스템 점검 중입니다.
  빠른 시일 내 복구하겠습니다.
  긴급 문의는 [전화번호]로 연락주세요."


==================================================
D-2. 대량 접수 오류 발생
==================================================

【증상】
여러 접수의 포인트가 잘못 차감됨

【즉시 조치】
1. 피해 범위 확인
   - 접수 내역 확인
   - 포인트 거래 내역 확인
   - 영향받은 거래처 파악

2. 시스템 접수 중단
   - 거래처들에게 공지
   - "임시 점검 중 - 접수 중단"

3. 오류 복구
   - 개발자에게 즉시 연락
   - 오류 원인 파악
   - 복구 방안 협의

4. 수동 환불 처리
   - 각 거래처별로 잘못 차감된 포인트 확인
   - 포인트 충전으로 환불
   - 사유: "시스템 오류 환불 - [날짜]"

5. 거래처 사과 및 보상
   - 전화/이메일로 사과
   - 환불 완료 안내
   - 필요시 보상 포인트 추가


==================================================
D-3. 거래처 계정 해킹 의심
==================================================

【증상】
거래처로부터 "제가 신청하지 않은 접수가 있어요" 연락

【즉시 조치】
1. 상황 확인
   - 접수 내역 확인
   - 접수 일시 및 내용 확인
   - IP 로그 확인 (가능한 경우)

2. 계정 긴급 조치
   - 해당 거래처 비활성화
   - 비밀번호 즉시 재설정
   - 진행중인 의심 접수 취소

3. 피해 조사
   - 의심 접수 목록 작성
   - 사용된 포인트 계산
   - 다른 거래처도 확인

4. 복구 및 보상
   - 의심 접수 모두 취소
   - 포인트 전액 환불
   - 추가 보상 포인트 지급

5. 재발 방지
   - 비밀번호 정책 강화
   - 로그인 알림 기능 검토
   - 보안 교육


==================================================
D-4. 대량 환불 요청
==================================================

【증상】
서비스 문제로 다수 거래처가 동시에 환불 요청

【즉시 조치】
1. 문제 파악
   - 환불 요청 사유 확인
   - 공통 문제점 파악
   - 서비스 제공 업체 연락

2. 영향 범위 확인
   - 영향받은 접수 목록 작성
   - 거래처 목록 작성
   - 환불 포인트 총액 계산

3. 우선순위 처리
   - 고액 거래처 우선
   - VIP 거래처 우선
   - 전화로 직접 사과

4. 일괄 환불 처리
   - 각 거래처별로 접수 취소
   - 포인트 환불
   - 사유: "서비스 문제 환불 - [날짜]"

5. 재발 방지 및 보상
   - 문제 원인 근본 해결
   - 보상 방안 수립
   - 신뢰 회복 노력


==================================================
D-5. 관리자 계정 분실
==================================================

【증상】
관리자가 비밀번호를 잊어버림

【조치 방법】
1. 데이터베이스 직접 접근
   - 개발자에게 연락
   - 비밀번호 재설정 요청

2. 복구 후
   - 즉시 비밀번호 변경
   - 보안 강화
   - 재발 방지

부록 E: 데이터 분석 활용 가이드

==================================================
E-1. 매출 증대 전략 수립
==================================================

애널리틱스 데이터를 활용하여 매출을 증대하는 방법입니다.

【1. 인기 상품 파악】

데이터 분석 → 상품 분석 탭
- 상품별 접수 건수 확인
- 가장 많이 팔리는 상품 파악

활용:
- 인기 상품 프로모션 강화
- 인기 상품 가격 정책 최적화
- 관련 상품 번들 판매

예시:
플레이스 유입이 60% 차지
→ 플레이스 + 영수증 리뷰 패키지 상품 기획
→ 10% 할인 프로모션

【2. VIP 거래처 관리】

데이터 분석 → 전체 개요 탭
- TOP 10 거래처 차트 확인
- 고액 거래처 파악

활용:
- VIP 거래처 특별 관리
- 전용 할인 혜택
- 우선 처리 서비스

예시:
월 100건 이상 거래처 3개 파악
→ 5% 추가 할인 제공
→ 전담 매니저 배정
→ 우선 처리 보장

【3. 비수기 대응】

데이터 분석 → 월간 분석 탭
- 12개월 추이 확인
- 비수기 파악

활용:
- 비수기 특별 프로모션
- 미리 선결제 시 보너스
- 장기 계약 유도

예시:
1~2월이 비수기로 확인
→ 12월에 선결제 프로모션
→ "1월 사용분 미리 결제 시 10% 보너스"

【4. 거래처 활성화】

거래처 관리 페이지
- 상태 필터: "활성" 선택
- 최근 접수가 없는 거래처 파악

활용:
- 휴면 거래처 발굴
- 재활성화 프로모션
- 맞춤 제안

예시:
2개월간 접수 없는 거래처
→ 전화 연락
→ "오랜만입니다. 특별 할인 제공합니다"
→ 20% 할인 쿠폰

【5. 시간대 최적화】

데이터 분석 → 일간 분석 탭
- 시간대별 접수 패턴 확인
- 피크 타임 파악

활용:
- 인력 배치 최적화
- 피크 타임 즉시 응대
- 시스템 리소스 관리

예시:
오후 2~5시가 피크 타임
→ 해당 시간에 인력 집중 배치
→ 빠른 처리로 고객 만족도 향상


==================================================
E-2. 거래처 세분화 전략
==================================================

거래처를 그룹화하여 맞춤 관리하는 방법입니다.

【세분화 기준】

1. 거래 규모별
   - 대형: 월 50건 이상
   - 중형: 월 20~50건
   - 소형: 월 20건 미만

2. 포인트 보유량별
   - 충분: 100만 P 이상
   - 보통: 50만~100만 P
   - 부족: 50만 P 미만

3. 활동 주기별
   - 정기: 매주 접수
   - 비정기: 월 1~2회 접수
   - 휴면: 2개월 이상 접수 없음

【세분화 활용】

대형 거래처:
- 전담 매니저 배정
- 특별 할인 (5~10%)
- 우선 처리
- 월간 리포트 제공

중형 거래처:
- 표준 서비스
- 계절 프로모션 제공
- 정기 소통

소형 거래처:
- 기본 서비스
- 셀프 서비스 유도
- 필요시만 연락

휴면 거래처:
- 재활성화 캠페인
- 특별 오퍼 제안
- 분기별 접촉


==================================================
E-3. 완료율 개선 전략
==================================================

접수 완료율을 높여 고객 만족도를 향상시키는 방법입니다.

【현황 파악】

데이터 분석 → 전체 개요
- 전체 완료율 확인
- 상품별 완료율 비교

목표 설정:
- 전체 완료율 95% 이상
- 모든 상품 완료율 90% 이상

【개선 방법】

1. 대기 시간 단축
   - 대기중 접수 즉시 진행중으로 변경
   - 작업자 충분 확보
   - 우선순위 시스템 운영

2. 진행 중 모니터링
   - 진행중 접수 매일 확인
   - 지연되는 건 원인 파악
   - 필요시 작업자 추가 투입

3. 품질 관리
   - 완료 기준 명확화
   - 작업자 교육 강화
   - 정기 품질 점검

4. AS 감소
   - AS 발생 원인 분석
   - 재발 방지 대책
   - 작업 가이드 개선

==================================================
F-1. 복잡한 검색 시나리오
==================================================

【시나리오 1: 특정 거래처의 미완료 고액 접수】

목적: 중요 거래처의 처리 필요 건 파악

단계:
1. 접수 내역 페이지
2. 검색: "[거래처명]" 입력
3. 상태: "대기중" 선택 (또는 "진행중")
4. 정렬: "포인트 높은순"
5. 결과: 해당 거래처의 고액 미완료 건만 표시

활용: 우선 처리 대상 선정

【시나리오 2: 이번 주 블로그 완료 건】

목적: 주간 블로그 성과 확인

단계:
1. 접수 내역 페이지
2. 상품 유형: "블로그 배포"
3. 상태: "완료"
4. 기간: "최근 7일"
5. 정렬: "최신순"
6. 결과: 이번 주 완료된 블로그 배포 목록

활용: 주간 보고서 작성

【시나리오 3: 포인트 부족 거래처】

목적: 충전 유도 대상 파악

단계:
1. 거래처 관리 페이지
2. 정렬: "포인트 적은순"
3. 상태: "활성"
4. 결과: 포인트 적은 활성 거래처 목록

활용:
- 포인트 부족 거래처에게 연락
- 충전 유도
- 프로모션 제안

【시나리오 4: 오늘의 긴급 처리 목록】

목적: 오늘 꼭 처리해야 할 접수 파악

단계:
1. 접수 내역 페이지
2. 상태: "대기중"
3. 기간: 사용 안함 (전체)
4. 정렬: "오래된순"
5. 결과: 오래 대기한 접수부터 표시

활용: 오래 대기한 건 우선 처리


==================================================
F-2. 거래처 행동 패턴 분석
==================================================

【분석 1: 접수 주기 파악】

방법:
1. 접수 내역에서 특정 거래처 검색
2. 정렬: "최신순"
3. 접수 날짜 간격 확인

패턴 예시:
- 매주 월요일 접수: 정기 거래처
- 불규칙: 비정기 거래처
- 월말 집중: 정산 타이밍 거래처

활용:
- 정기 거래처: 자동 알림 설정 검토
- 비정기 거래처: 접촉 빈도 증가
- 월말 집중: 미리 준비

【분석 2: 선호 상품 파악】

방법:
1. 접수 내역에서 거래처 검색
2. 상품 유형별로 건수 세기

예시:
강남 카페:
- 플레이스 유입: 30건 (60%)
- 영수증 리뷰: 15건 (30%)
- 카카오맵: 5건 (10%)
- 블로그: 0건 (0%)

활용:
- 주력 상품 추가 할인
- 미사용 상품 샘플 제공
- 맞춤 패키지 제안

【분석 3: 포인트 사용 패턴】

방법:
1. 포인트 관리 페이지
2. 특정 거래처 검색
3. 충전 주기 확인

패턴:
- 정기 충전: 안정적 거래처
- 불규칙 충전: 프로젝트성 거래처
- 미충전: 초기 포인트만 사용

활용:
- 정기 충전: 자동 충전 제안
- 불규칙: 선결제 할인 제안
- 미충전: 추가 계약 제안

부록 G: 보고서 작성 가이드

==================================================
G-1. 일일 보고서
==================================================

【보고서 구성】

제목: [날짜] 일일 운영 현황

1. 요약
   - 오늘 접수: N건
   - 오늘 매출: N P
   - 신규 거래처: N개
   - AS 처리: N건

2. 상세
   - 상품별 접수 현황
   - 처리 완료 건수
   - 대기중 건수
   - 주요 이슈

3. 특이사항
   - 고액 접수
   - 신규 거래처
   - AS 발생 건

【작성 방법】

1. 데이터 수집:
   - 대시보드에서 기본 통계
   - 데이터 분석 → 일간 분석
   - 접수 내역 필터링

2. 보고서 작성:
   - 워드 또는 엑셀
   - 템플릿 활용
   - 차트 캡처 첨부

3. 배포:
   - 경영진 이메일 발송
   - 팀 메신저 공유


==================================================
G-2. 주간 보고서
==================================================

【보고서 구성】

제목: [년도-주차] 주간 운영 리포트
예: 2024-45주차 주간 운영 리포트

1. 주간 요약
   - 이번 주 접수: N건 (지난 주 대비 +N%)
   - 이번 주 매출: N P (지난 주 대비 +N%)
   - 신규 거래처: N개
   - 완료율: N%

2. 상품별 분석
   - 플레이스 유입: N건, N P
   - 영수증 리뷰: N건, N P
   - 카카오맵 리뷰: N건, N P
   - 블로그 배포: N건, N P

3. TOP 5 거래처
   - 접수 건수 기준
   - 매출 기준

4. 이슈 및 대응
   - 발생한 문제
   - 처리 내용
   - 재발 방지 대책

5. 다음 주 계획
   - 목표 설정
   - 주요 일정
   - 개선 과제

【작성 방법】

1. 데이터 수집:
   - 데이터 분석 → 주간 분석 탭
   - 접수 내역 다운로드 (최근 7일)
   - AS 내역 확인

2. 엑셀 분석:

==================================================
J-1. 만족도 측정
==================================================

【측정 방법】

1. 직접 문의 (전화)
   - 월 1회 주요 거래처에게 전화
   - "서비스 만족도는 어떠신가요?"
   - "개선이 필요한 부분은?"

2. AS 발생률 분석
   - 데이터 분석 → 전체 개요
   - AS 발생률 확인
   - 기준: 5% 미만 양호

3. 재주문율 분석
   - 거래처별 접수 주기 확인
   - 정기적 주문 = 높은 만족도
   - 휴면 = 낮은 만족도

【만족도 지표】

높은 만족도:
- 정기적 주문 (주 1회 이상)
- AS 발생 없음
- 추가 계약 체결
- 추천 고객 유입

보통 만족도:
- 비정기 주문 (월 1~2회)
- 간혹 AS 발생
- 계약 유지

낮은 만족도:
- 주문 감소
- 빈번한 AS
- 환불 요청
- 계약 해지 의사

L-3. 엑셀 고급 분석

【다운로드한 엑셀 파일 활용】

【분석 1: 거래처별 상품 선호도】

1. 피벗 테이블 생성

   - 열: 상품유형

   - 값: 접수 건수 (개수)

2. 라인 차트 생성
   - X축: 날짜
   - Y축: 건수
   - 추세선 추가

【분석 3: 평균 단가 계산】

1. 피벗 테이블 생성

     * 사용포인트 (평균)
     * 접수 건수 (개수)

2. 상품별 평균 단가 확인
   - 가격 정책 수립에 활용

【분석 4: 완료율 계산】

1. 피벗 테이블 생성
   - 행: 상품유형 또는 거래처
   - 열: 상태
   - 값: 접수 건수 (개수)

2. 계산 필드 추가
   - 완료율 = 완료 / (전체) * 100

【분석 5: 시간대별 접수】

1. 접수일시에서 시간 추출
   - 엑셀 함수: =HOUR(접수일시)
   
2. 피벗 테이블 생성
   - 행: 시간
   - 값: 건수

3. 막대 차트 생성
   - 피크 타임 파악

N-2. 데이터 관계 이해
==================================================

【관계 1: 거래처 ↔ 접수】
- 하나의 거래처는 여러 접수 가능
- 각 접수는 하나의 거래처에 속함
- client_id로 연결

조회 예시:
특정 거래처의 모든 접수 찾기
→ 접수 내역에서 거래처 검색

【관계 2: 거래처 ↔ 포인트 거래】
- 하나의 거래처는 여러 포인트 거래 가능
- 각 거래는 하나의 거래처에 속함
- client_id로 연결

조회 예시:
특정 거래처의 포인트 내역
→ 포인트 관리에서 거래처 검색

【관계 3: 거래처 ↔ AS 요청】
- 하나의 거래처는 여러 AS 요청 가능
- 각 AS는 특정 접수와 연결
- client_id, submission_id로 연결

조회 예시:
특정 거래처의 AS 요청
→ AS 관리에서 거래처로 필터

【관계 4: 거래처 ↔ 가격 설정】
- 하나의 거래처는 4개 상품 가격 가짐
- 각 가격은 하나의 거래처에 속함
- client_id, category_id로 연결

설정 방법:
→ 거래처 관리 → 가격 설정

==================================================
O-1. 주요 KPI 정의
==================================================

【운영 효율성 KPI】

1. 평균 처리 시간
   - 정의: 접수부터 완료까지 평균 일수
   - 목표: 3일 이내
   - 측정: 데이터 분석 → 전체 개요
   - 개선: 프로세스 최적화, 인력 증원

2. 완료율
   - 정의: 완료 건수 / 전체 건수 * 100
   - 목표: 95% 이상
   - 측정: 데이터 분석 → 전체 개요
   - 개선: 미완료 건 원인 분석 및 해결

3. AS 발생률
   - 정의: AS 건수 / 완료 건수 * 100
   - 목표: 5% 미만
   - 측정: 데이터 분석 → 전체 개요
   - 개선: 품질 관리 강화

【매출 성장 KPI】

4. 월간 매출 (포인트 사용)
   - 정의: 한 달간 사용된 총 포인트
   - 목표: 전월 대비 10% 증가
   - 측정: 데이터 분석 → 월간 분석
   - 개선: 거래처 확대, 상품 다양화

5. 접수 건수
   - 정의: 한 달간 총 접수 건수
   - 목표: 전월 대비 15% 증가
   - 측정: 데이터 분석 → 월간 분석
   - 개선: 마케팅 강화, 프로모션

6. 평균 객단가
   - 정의: 총 매출 / 총 접수 건수
   - 목표: [N] P 이상
   - 측정: 수동 계산 또는 엑셀
   - 개선: 고단가 상품 판매 증대

【거래처 관리 KPI】

7. 활성 거래처 비율
   - 정의: 활성 거래처 / 전체 거래처 * 100
   - 목표: 80% 이상
   - 측정: 거래처 관리 페이지
   - 개선: 휴면 거래처 재활성화

8. 신규 거래처 (월간)
   - 정의: 한 달간 신규 등록 거래처 수
   - 목표: 월 5개 이상
   - 측정: 데이터 분석 → 월간 분석
   - 개선: 영업 활동 강화

9. 포인트 회전율
   - 정의: 포인트 사용 / 포인트 충전 * 100
   - 목표: 80~100%
   - 측정: 데이터 분석 → 전체 개요
   - 개선: 사용 촉진 프로모션

【고객 만족도 KPI】

10. 재주문율
    - 정의: 재주문 거래처 / 전체 거래처 * 100
    - 목표: 70% 이상
    - 측정: 수동 계산
    - 개선: 서비스 품질 향상

11. AS 처리 시간
    - 정의: AS 신청부터 처리까지 평균 시간
    - 목표: 4시간 이내
    - 측정: 수동 계산
    - 개선: 빠른 응대 체계

12. 추천 고객 수
    - 정의: 기존 거래처 추천으로 유입된 신규 거래처
    - 목표: 월 1개 이상
    - 측정: 수동 기록
    - 개선: 추천 인센티브 제도

확인 방법:
→ 대시보드 통계 카드
→ 데이터 분석 → 일간 분석

【주간 모니터링】
- 이번 주 접수
- 이번 주 매출
- 일평균 접수
- 주간 완료율

확인 방법:
→ 데이터 분석 → 주간 분석
→ 접수 내역 엑셀 분석

【월간 모니터링】
- 월간 총 접수
- 월간 총 매출
- 신규 거래처
- 월간 완료율
- AS 발생률

확인 방법:
→ 데이터 분석 → 월간 분석
→ 전체 개요 탭

【KPI 대시보드 만들기】

   - 주차별로 기록
   - 막대 차트 생성

   - 추세선 분석

4. 시트 4: 종합 대시보드
   - 주요 KPI 한눈에

부록 Q: 트러블슈팅 상세 가이드
============================================================================

==================================================
Q-1. 화면이 안 보이는 경우
==================================================

【증상 1: 통계 카드가 비어있음】
원인: 데이터 로딩 중
해결: 5초 대기 후 자동 표시

【증상 2: 차트가 안 보임】
원인: JavaScript 오류
해결:
1. F5로 새로고침
2. 브라우저 캐시 삭제
3. 다른 브라우저 시도

【증상 3: 목록이 비어있음】
원인: 필터 설정 문제
해결:
1. 모든 필터를 "전체"로 변경
2. 검색어 삭제
3. 새로고침

【증상 4: 버튼이 작동 안함】
원인: 네트워크 오류
해결:
1. 인터넷 연결 확인
2. 페이지 새로고침
3. 로그아웃 후 재로그인


==================================================
Q-2. 데이터가 이상한 경우
==================================================

【증상 1: 포인트 이중 차감】
확인:
1. 포인트 거래 내역 확인
2. 중복 거래 찾기

해결:
1. 중복 차감 확인 시
2. 해당 금액 즉시 환불
3. 거래처에게 사과 및 설명
4. 개발자에게 버그 보고

【증상 2: 접수가 두 번 들어감】
확인:
1. 접수 내역에서 검색
2. 동일 시간대 중복 확인

해결:
1. 중복 접수 중 하나 취소
2. 포인트 환불
3. 거래처에게 안내

【증상 3: 통계 숫자가 맞지 않음】
확인:
1. 수동으로 접수 세어보기
2. 실제 데이터와 비교

해결:
1. 새로고침 (캐시 문제)
2. 다른 브라우저에서 확인
3. 여전히 불일치 시 개발자 보고


==================================================
Q-3. 성능 문제 해결
==================================================

【증상: 페이지가 느림】

원인 1: 데이터 과다
- 접수 내역이 수천 건 이상
- 필터 없이 전체 조회

해결:
- 기간 필터 사용
- "최근 30일"로 제한
- 엑셀 다운로드 후 분석

원인 2: 브라우저 캐시
- 오래된 캐시 데이터

해결:
- Ctrl + Shift + Delete
- 브라우저 캐시 삭제
- 쿠키는 유지 권장

원인 3: 네트워크 속도
- 느린 인터넷

해결:
- WiFi 재연결
- 유선 연결 권장
- 다른 네트워크 시도
