[1부] 시스템 시작하기

────────────────────────────────────────────────────────────────────────────
1.1 로그인 방법
────────────────────────────────────────────────────────────────────────────

■ 초기 접속 정보
───────────────────────────────────────────────────────────────────────────
관리자로부터 다음 정보를 받으셨을 것입니다:

▶ 아이디: 관리자가 설정한 귀사 전용 아이디
▶ 비밀번호: 관리자가 설정한 초기 비밀번호
▶ 초기 포인트: 계약에 따른 초기 충전 포인트

■ 로그인 절차 (단계별)
───────────────────────────────────────────────────────────────────────────
1단계: 브라우저 열기
   - 권장 브라우저: Chrome, Edge, Safari (최신 버전)
   - Internet Explorer는 지원하지 않습니다

2단계: 시스템 주소 입력
   - 주소창에 시스템 URL 입력
   - 북마크(즐겨찾기)에 저장해두시면 편리합니다

3단계: 로그인 화면에서 입력
   - "거래처" 탭이 선택되어 있는지 확인 (중요!)
   - 아이디 입력 (대소문자 구분)
   - 비밀번호 입력

4단계: 로그인 버튼 클릭
   - "로그인" 버튼을 클릭하면 대시보드로 이동합니다

⚠ 주의사항:
   - "관리자" 탭이 아닌 "거래처" 탭에서 로그인해야 합니다
   - 비밀번호를 3회 이상 틀리면 관리자에게 문의하세요
   - 로그인 정보는 타인과 공유하지 마세요

■ 로그인 문제 해결
───────────────────────────────────────────────────────────────────────────
문제 1: "아이디 또는 비밀번호가 올바르지 않습니다"
   → 해결: 아이디/비밀번호 대소문자 확인
   → 해결: 공백이 입력되지 않았는지 확인
   → 해결: 관리자에게 정보 재확인 요청

문제 2: "계정이 비활성화되었습니다"
   → 해결: 관리자에게 계정 활성화 요청

문제 3: 로그인 후 바로 로그아웃됨
   → 해결: 브라우저 쿠키 설정 확인
   → 해결: 시크릿 모드가 아닌 일반 모드 사용

■ 로그아웃 방법
───────────────────────────────────────────────────────────────────────────
▶ 화면 우측 상단의 "로그아웃" 버튼 클릭
▶ 작업 완료 후 반드시 로그아웃하세요 (보안)
▶ 세션 유효시간: 24시간 (자동 로그아웃)

│                                                                 │

■ 화면 구성 요소 설명
───────────────────────────────────────────────────────────────────────────
1) 상단 헤더
   - 회사명: 귀사의 등록된 회사명 표시
   - 보유 포인트: 현재 사용 가능한 포인트 (실시간 업데이트)
   - 로그아웃 버튼: 안전하게 시스템 종료

2) 통계 카드
   - 보유 포인트: 현재 잔여 포인트
   - 총 접수 건수: 지금까지 접수한 모든 상품의 합계

3) 이용 가능한 상품 섹션
   - 관리자가 활성화한 상품만 표시됩니다
   - 각 상품 카드에는 이름, 설명, 단가가 표시됩니다
   - "접수하기" 버튼을 클릭하면 해당 상품 접수 페이지로 이동

4) 최근 접수 내역
   - 최근 5건의 접수 기록을 시간순으로 표시
   - 날짜, 상품명, 사용 포인트를 한눈에 확인

■ 좌측 메뉴 (네비게이션)
───────────────────────────────────────────────────────────────────────────
메뉴 항목:
├─ 대시보드 (홈)
├─ 상품 접수
├─ 접수 내역
├─ 포인트 거래 내역
└─ AS 신청

각 메뉴를 클릭하면 해당 페이지로 이동합니다.

■ 반응형 디자인 (모바일 사용)
───────────────────────────────────────────────────────────────────────────
▶ PC, 태블릿, 스마트폰 모두 지원
▶ 모바일에서는 좌측 메뉴가 햄버거 버튼(☰)으로 변경
▶ 화면 크기에 맞춰 자동으로 레이아웃 조정


────────────────────────────────────────────────────────────────────────────
1.3 포인트 시스템 이해하기
────────────────────────────────────────────────────────────────────────────

■ 포인트란?
───────────────────────────────────────────────────────────────────────────
포인트는 상품 접수 시 사용되는 가상 화폐입니다.
▶ 1 포인트 = 1원 개념 (단, 실제 환급은 불가)
▶ 관리자가 계약에 따라 충전해줍니다
▶ 상품 접수 시 자동으로 차감됩니다

■ 포인트 확인 방법
───────────────────────────────────────────────────────────────────────────
방법 1: 대시보드 상단
   - 로그인하면 가장 먼저 보이는 위치
   - 실시간으로 업데이트됩니다

방법 2: 상품 접수 페이지
   - 접수 폼 하단의 "포인트 계산" 섹션
   - 필요 포인트와 보유 포인트를 비교 표시

방법 3: 포인트 거래 내역 페이지
   - 좌측 메뉴 > "포인트 거래 내역" 클릭
   - 모든 충전/사용 기록을 상세히 확인

■ 포인트 충전
───────────────────────────────────────────────────────────────────────────
⚠ 거래처는 직접 충전할 수 없습니다!
▶ 관리자에게 충전 요청 (전화, 이메일, AS 신청)
▶ 계약 금액 입금 후 충전 처리
▶ 충전 시 포인트 거래 내역에 자동 기록

충전 기록 확인:
   구분: "충전"
   금액: +xxxxx P
   설명: "계약금 입금" 등

■ 포인트 사용
───────────────────────────────────────────────────────────────────────────
▶ 상품 접수 시 자동으로 차감
▶ 차감 공식: (단가 × 수량) 또는 (단가 × 일수 × 일타수)
▶ 포인트 부족 시 접수 불가능

사용 기록 확인:
   구분: "차감"
   금액: -xxxxx P
   설명: "플레이스 유입 접수" 등

■ 포인트 관리 꿀팁
───────────────────────────────────────────────────────────────────────────
✓ 포인트 잔액을 항상 확인하고 접수하세요
✓ 대량 접수 전에 충분한 포인트가 있는지 미리 체크
✓ 포인트 부족 시 미리 충전 요청 (처리 시간 고려)
✓ 포인트 거래 내역을 정기적으로 확인하여 이상 거래 체크

────────────────────────────────────────────────────────────────────────────
2.1 플레이스 유입 접수 (네이버 플레이스)
────────────────────────────────────────────────────────────────────────────

■ 서비스 개요
───────────────────────────────────────────────────────────────────────────
네이버 플레이스에 고객 유입을 발생시켜 노출도와 순위를 높이는 서비스입니다.

▶ 대상: 네이버 플레이스에 등록된 업체
▶ 효과: 검색 순위 상승, 방문자 수 증가, 인지도 향상
▶ 최소 단위: 일 100타 × 최소 3일
▶ 최대 기간: 7일

■ 접수 전 준비사항
───────────────────────────────────────────────────────────────────────────
✓ 네이버 플레이스 등록 완료 (필수)
✓ 모바일 플레이스 URL 준비 (https://m.place.naver.com/... 형태)
✓ 필요 포인트 계산: 단가 × 일타수 × 구동일수
✓ 보유 포인트 충분한지 확인

예시 계산:
   단가: 50원
   일 접수량: 200타
   구동일수: 5일
   필요 포인트 = 50 × 200 × 5 = 50,000 P

■ 접수 절차 (상세)
───────────────────────────────────────────────────────────────────────────
1단계: 접수 페이지 진입
   방법 1: 대시보드 > "플레이스 유입" 카드 > "접수하기" 버튼
   방법 2: 좌측 메뉴 > "상품 접수" > "플레이스 유입 접수" 선택

2단계: 접수 정보 입력
   ┌─────────────────────────────────────────────┐
   │ ■ 접수 정보                                 │
   ├─────────────────────────────────────────────┤
   │                                             │
   │ 업체명 * [                    ]             │
   │ → 플레이스에 등록된 업체명 정확히 입력     │
   │    예: "강남 커피숍", "홍대 맛집"          │
   │                                             │
   │ 플레이스 URL (모바일 링크) *                │
   │ [                                         ] │
   │ → 반드시 모바일 URL 사용!                  │
   │    올바른 예: https://m.place.naver.com/   │
   │              restaurant/12345678           │
   │    잘못된 예: https://place.naver.com/...  │
   │                                             │
   │ 일 접수량 * [ 100 ] 타                      │
   │ → 최소 100타 이상                          │
   │ → 권장: 100~500타 (자연스러운 유입)       │
   │                                             │
   │ 구동일수 * [ 3 ] 일                         │
   │ → 최소 3일 ~ 최대 7일                      │
   │ → 권장: 5~7일 (효과 극대화)               │
   │                                             │
   │ 비고                                        │
   │ [                                         ] │
   │ → 특별 요청사항 입력 (선택사항)           │
   │                                             │
   └─────────────────────────────────────────────┘

3단계: 포인트 계산 확인
   ┌─────────────────────────────────────────────┐
   │ ■ 포인트 계산                               │
   ├─────────────────────────────────────────────┤
   │                                             │
   │ 단가:           50 P                        │
   │ 일 접수량:     200타                        │
   │ 구동일수:       5일                         │
   │ ────────────────────────────────────       │
   │ 총 차감 포인트: 50,000 P                    │
   │ 보유 포인트:   100,000 P ✓                 │
   │                                             │
   └─────────────────────────────────────────────┘

   ⚠ 주의:
   - 빨간색으로 표시되면 포인트 부족
   - 녹색으로 표시되면 접수 가능

4단계: 최종 확인 및 접수
   - 모든 정보를 다시 한 번 확인
   - "접수하기" 버튼 클릭
   - 확인 메시지가 나오면 성공!

5단계: 접수 완료 후
   - 자동으로 대시보드로 이동
   - 포인트가 즉시 차감됨
   - "접수 내역"에서 확인 가능

■ 모바일 URL 찾는 방법
───────────────────────────────────────────────────────────────────────────
방법 1: 네이버 앱에서 찾기
   1. 네이버 앱 실행
   2. 귀사 업체 검색
   3. 플레이스 페이지 진입
   4. 공유 버튼 > 링크 복사
   5. 복사된 URL 사용

방법 2: 모바일 브라우저에서 찾기
   1. 모바일 크롬/사파리 실행
   2. m.naver.com 접속
   3. 업체 검색 후 플레이스 진입
   4. 주소창의 URL 복사

방법 3: PC에서 찾기
   1. place.naver.com에서 업체 검색
   2. URL의 숫자 부분 복사 (예: 12345678)
   3. https://m.place.naver.com/restaurant/12345678 형태로 조합

■ 입력 시 주의사항
───────────────────────────────────────────────────────────────────────────
✗ 잘못된 예시:
   - 업체명: "ㅇㅇ 카페" (약자 사용)
   - URL: PC 버전 URL 사용
   - 일 접수량: 50타 (최소 미만)
   - 구동일수: 10일 (최대 초과)

✓ 올바른 예시:
   - 업체명: "강남구 역삼동 커피숍"
   - URL: https://m.place.naver.com/restaurant/12345678
   - 일 접수량: 200타
   - 구동일수: 5일

■ 자주 묻는 질문
───────────────────────────────────────────────────────────────────────────
Q: 일 접수량을 많이 할수록 좋나요?
A: 아니요. 너무 많으면 부자연스러워 역효과가 날 수 있습니다.
   권장량은 100~300타 정도입니다.

Q: 구동일수는 며칠이 적당한가요?
A: 효과를 보려면 최소 5일 이상을 권장합니다.

Q: URL이 계속 오류가 나요.
A: 반드시 m.place.naver.com으로 시작하는 모바일 URL을 사용하세요.

Q: 접수 후 언제부터 시작되나요?
A: 관리자 승인 후 영업일 기준 1~2일 내 시작됩니다.


────────────────────────────────────────────────────────────────────────────
2.2 영수증 리뷰 접수
────────────────────────────────────────────────────────────────────────────

■ 서비스 개요
───────────────────────────────────────────────────────────────────────────
실제 영수증을 기반으로 한 진정성 있는 리뷰 작성 서비스입니다.

▶ 대상: 오프라인 매장 운영 업체 (음식점, 카페, 서비스업 등)
▶ 효과: 신뢰도 높은 리뷰 확보, 별점 상승
▶ 최소 단위: 30타
▶ 특징: 사업자등록증 + 샘플 영수증 + 사진 필수

■ 접수 전 준비사항
───────────────────────────────────────────────────────────────────────────
필수 준비물:
✓ 사업자등록증 또는 샘플 영수증 파일 (이미지 or PDF)
✓ 매장 사진 최소 1장 이상 (JPG, PNG 등)
✓ 네이버 플레이스 URL
✓ 필요 포인트 계산: 단가 × 총 타수

파일 준비 가이드:
▶ 사업자등록증: 명확하게 읽을 수 있도록 스캔 또는 촬영
▶ 샘플 영수증: 실제 영수증 사진 (업체명, 금액 확인 가능)
▶ 매장 사진: 실내외, 메뉴, 제품 등 다양하게 준비
▶ 파일 크기: 각 파일 최대 10MB
▶ 파일 형식: 이미지(JPG, PNG), PDF

✓ 샘플 영수증은 최근 것으로
   - 너무 오래된 영수증은 사용 불가
   - 업체명, 날짜, 금액이 명확해야 함

✓ 사진은 다양하게 준비
   - 매장 외관, 내부, 메뉴, 음식 등

   - 실제 방문한 느낌이 나도록

오류: "지원하지 않는 파일 형식"
   → 해결: JPG, PNG, PDF 형식만 지원
   → 변환: HEIC → JPG 변환 필요 (아이폰 사진)

오류: "업로드에 실패했습니다"
   → 해결: 인터넷 연결 확인
   → 해결: 브라우저 새로고침 후 재시도

A: 참고사항에 요청하시면 반영됩니다.

────────────────────────────────────────────────────────────────────────────
2.4 블로그 배포 접수
────────────────────────────────────────────────────────────────────────────

■ 서비스 개요
───────────────────────────────────────────────────────────────────────────
네이버 블로그에 홍보 콘텐츠를 작성하여 온라인 노출을 극대화하는 서비스입니다.

▶ 대상: 모든 업종 (특히 검색 노출이 중요한 업체)
▶ 효과: 네이버 검색 상위 노출, 브랜드 인지도 상승, 유입 증가
▶ 최소 단위: 일 1타
▶ 최대 단위: 총 30타
▶ 특징: 3가지 배포 타입, 2가지 콘텐츠 타입 선택 가능

■ 배포 타입 설명
───────────────────────────────────────────────────────────────────────────
1) 리뷰어형 (reviewer)
   설명: 실제 체험 후기 형태의 블로그 포스팅
   특징: 신뢰도 높음, 자연스러운 홍보
   적합한 업종: 음식점, 카페, 서비스업, 제품 판매
   예시: "○○카페 다녀온 후기, 분위기와 맛 모두 만족!"

2) 영상형 (video)
   설명: 영상 콘텐츠를 포함한 블로그 포스팅
   특징: 시각적 임팩트, 높은 체류시간
   적합한 업종: 요리, 미용, 운동, 제품 소개
   예시: "○○제품 언박싱 영상과 사용 후기"

3) 자동화형 (automation)
   설명: 키워드 최적화된 자동 생성 콘텐츠
   특징: 빠른 배포, 대량 노출
   적합한 업종: 정보 제공이 필요한 업종
   예시: "○○지역 맛집 추천 베스트 5"

■ 콘텐츠 타입 설명
───────────────────────────────────────────────────────────────────────────
1) 리뷰형 (review)
   - 실제 방문/사용 후기 형태
   - 주관적 감상과 평가 포함
   - 별점, 추천 여부 등 리뷰 요소 포함

2) 정보형 (info)
   - 객관적 정보 제공 형태
   - 업체 소개, 메뉴, 가격, 위치 등 안내
   - 검색 유입에 최적화

■ 접수 전 준비사항
───────────────────────────────────────────────────────────────────────────
✓ 업체의 네이버 플레이스 또는 홈페이지 URL
✓ 강조하고 싶은 키워드 목록
✓ 배포 타입과 콘텐츠 타입 결정
✓ 일별 타수와 기간 계획
✓ 필요 포인트 계산

포인트 계산 예시:

   일 타수: 2타
   총 일수: 10일
   필요 포인트 = 10,000 × (2 × 10) = 200,000 P

■ 효과적인 블로그 배포 전략
───────────────────────────────────────────────────────────────────────────
전략 1: 꾸준한 배포
   - 한 번에 많이보다는 꾸준히
   - 일 2타 × 10일 권장
   - 자연스러운 노출 증가

전략 2: 키워드 다양화
   - 매 접수마다 다른 키워드 사용
   - 경쟁 키워드 + 롱테일 키워드 조합
   - 지역, 업종, 특징 키워드 골고루

전략 3: 타입 조합
   - 리뷰어형 + 정보형 혼합
   - 다양한 관점에서 노출
   - 검색 결과 상위 점유

───────────────────────────────────────────────────────────────────────────
Q: 블로그는 어느 블로그에 작성되나요?

Q: 작성된 포스팅을 확인할 수 있나요?
A: 네, 관리자가 리포트로 제공합니다.

Q: 키워드를 꼭 입력해야 하나요?
A: 선택사항이지만, 입력하면 더 효과적입니다.

Q: 한 번에 30타를 다 써도 되나요?
A: 가능하지만, 분산 배포가 더 효과적입니다.

Q: 배포 타입은 어떤 걸 선택해야 하나요?
A: 업종과 목적에 따라 다릅니다. 일반적으로 리뷰어형을 추천합니다.

3.3 AS 신청하기
────────────────────────────────────────────────────────────────────────────

■ AS 신청이란?
───────────────────────────────────────────────────────────────────────────
서비스가 약속된 기준에 미달했을 때 재작업 또는 환불을 요청하는 기능입니다.

▶ 대상: 서비스 미달 20% 이상인 경우
▶ 결과: 재작업 또는 포인트 환불
▶ 처리: 관리자 검토 후 승인

■ AS 신청 가능한 경우
───────────────────────────────────────────────────────────────────────────
✓ 약속된 수량보다 20% 이상 부족한 경우
   예: 100타 접수 → 실제 70타만 진행 (30% 미달)

✓ 서비스 품질이 현저히 낮은 경우
   예: 블로그 포스팅이 엉터리로 작성됨

✓ 서비스가 전혀 진행되지 않은 경우
   예: 승인 후 2주 경과했는데 아무 작업 없음

✓ 접수 오류로 포인트만 차감된 경우

■ AS 신청 불가능한 경우
───────────────────────────────────────────────────────────────────────────
✗ 단순 변심
✗ 효과가 없다는 주관적 판단
✗ 20% 미만의 미달
✗ 귀사의 정보 오류로 인한 문제

■ 접수 절차
───────────────────────────────────────────────────────────────────────────
1단계: AS 신청 페이지 진입
   방법: 좌측 메뉴 > "AS 신청" 클릭

2단계: 신청 정보 입력

   │ │ 미달률: 30%                          │   │

3단계: 상세 설명 작성 요령
   구체적으로 작성할 내용:
   ▶ 언제 문제를 발견했는지
   ▶ 어떤 부분이 미달인지
   ▶ 확인 가능한 증거 (있다면)
   ▶ 원하는 해결 방법 (재작업 or 환불)

   작성 예시:
   "1월 15일에 플레이스 유입 1000타를 접수했습니다.
   1월 25일 확인 결과 700타만 진행되었고,
   네이버 플레이스 통계에서도 확인됩니다.

────────────────────────────────────────────────────────────────────────────

■ 포인트 충전 계획
───────────────────────────────────────────────────────────────────────────
추천 충전 방식:

방식 1: 월 단위 충전
   장점: 계획적 사용, 예산 관리 용이
   방법: 매월 초 1회 충전
   적합 대상: 정기 이용 업체

방식 2: 분기 단위 충전
   장점: 할인 혜택 (있는 경우)
   방법: 분기 초 대량 충전
   적합 대상: 대형 업체

방식 3: 수시 충전
   장점: 필요한 만큼만
   방법: 포인트 부족 시마다
   적합 대상: 비정기 이용 업체

4.3 자주 묻는 질문 (FAQ)
────────────────────────────────────────────────────────────────────────────

■ 로그인 및 계정
───────────────────────────────────────────────────────────────────────────
Q1. 비밀번호를 잊어버렸어요.
A: 거래처는 직접 변경할 수 없습니다. 관리자에게 초기화를 요청하세요.

Q2. 여러 명이 같은 계정을 사용해도 되나요?
A: 가능하지만 보안상 권장하지 않습니다.

Q3. 로그인 상태가 자동으로 풀려요.
A: 세션 만료(24시간)이거나 브라우저 설정 문제일 수 있습니다.

■ 포인트
───────────────────────────────────────────────────────────────────────────
Q4. 포인트는 어떻게 충전하나요?
A: 관리자에게 요청하세요. 입금 후 충전 처리됩니다.

Q5. 포인트 유효기간이 있나요?
A: 일반적으로 없지만, 계약에 따라 다를 수 있습니다.

Q6. 남은 포인트를 환불받을 수 있나요?
A: 계약 조건에 따릅니다. 관리자에게 문의하세요.


────────────────────────────────────────────────────────────────────────────
4.4 문제 해결 가이드
────────────────────────────────────────────────────────────────────────────

■ 로그인 문제
───────────────────────────────────────────────────────────────────────────
증상: 로그인이 안 돼요
해결 1: 아이디/비밀번호 대소문자 확인
해결 2: "거래처" 탭 선택 확인
해결 3: 브라우저 쿠키 허용 설정

■ 접수 오류
───────────────────────────────────────────────────────────────────────────
증상: "포인트가 부족합니다" 오류
해결 1: 대시보드에서 포인트 확인
해결 2: 포인트 충전 요청

증상: 파일 업로드가 안 돼요
해결 1: 파일 크기 10MB 이하 확인
해결 2: 파일 형식 확인 (JPG, PNG, PDF만)
